[페이지 로드](before-starting#load-page) 이벤트는 웹 애플리케이션을 방문한 최종 사용자가 페이지로드를 완료했을 때 발생하는 이벤트입니다. <b>페이지 로드</b> 이벤트를 히트맵 차트 형태로 제공하고 각 이벤트에 대한 상세 정보를 타임라인 형식으로 확인할 수 있습니다. 이를 통해 웹 페이지를 로드하는 데 장애를 발생시키는 요소를 파악할 수 있습니다. 다음의 정보를 포함합니다.

* **페이지 로드** 이벤트 구간별 세부 시간 정보

* 로드한 리소스의 세부 정보

* 브라우저 에러 데이터

## 페이지 로드 이벤트 목록 확인 {#pageload-events-list}

단일 페이지별로 로드 이벤트를 분석할 수 있습니다. 페이지 로드 성능을 분석하기 위해 히트맵과 다양한 지표를 제공합니다.

<ImgLang img='browser-analysis-pageload-hitmap.png' desc='페이지 로드 히트맵' />

![지시선 1](/img/number-01.png) <Cmdname sid="rumPageLoadHitmap" className="uitext" /> 위젯의 차트 영역을 드래그하세요. 드래그한 영역의 페이지 로드 이벤트 목록을 화면 아래에 ![지시선 2](/img/number-02.png) <Cmdname sid="page" className="uitext" /> <Cmdname sid="list" className="uitext" />으로 불러오고, [페이지 그룹](before-starting#page-group)별로 분류해 화면 오른쪽에 ![지시선 4](/img/number-04.png) <Cmdname sid="page_group" className="uitext" /> <Cmdname sid="list" className="uitext" />으로 불러옵니다. ![지시선 3](/img/number-03.png) <Cmdname sid="page_url" className="uitext" />에서 원하는 페이지 목록을 검색할 수 있습니다.

### ![](/img/number-01.png) 페이지 로드 히트맵 {#pageload-hitmap}

<Cmdname sid="rumPageLoadHitmap" className="uitext" /> 위젯은 시간에 따른 <b>페이지 로드</b>의 응답시간을 분포도 차트로 표현합니다. 

* 가로축은 <b>페이지 로드</b> 이벤트의 종료 시간입니다.

* 세로축은 <b>페이지 로드</b> 이벤트의 경과 시간입니다.

* <span class='lowfg'>하늘색</span> → <span class='normal'>파란색</span> → <span class='highfg'>남색</span> 순으로 정상 <b>페이지 로드</b> 이벤트를 표현합니다.

* <span class='logwr'>노란색</span> → <span class='slow'>주황색</span> → <span class='vslow'>빨간색</span> 순으로 에러 발생 <b>페이지 로드</b> 이벤트를 표현합니다. 에러 <b>페이지 로드</b> 이벤트는 페이지를 불러오는 중 브라우저 에러가 발생한 경우입니다.


### ![](/img/number-02.png) 페이지 목록 {#page-list}

<Cmdname sid="rumPageLoadHitmap" className="uitext" /> 위젯에서 드래그한 영역의 <b>페이지 로드</b> 이벤트들이 <Cmdname sid="page" className="uitext" /> <Cmdname sid="list" className="uitext" /> 위젯의 테이블에 나타납니다. 목록에서 각 항목의 가장 왼쪽에 <LinkImage img="ico-detail-view.svg" className="ico" desc="상세 보기 아이콘" /> 버튼을 선택하면 <Cmdname sid="page_load" className="uitext" /> <Cmdname sid="detail" className="uitext" /> 창이 나타납니다.

<Cmdname sid="page" className="uitext" /> <Cmdname sid="list" className="uitext" /> 위젯의 테이블에는 다음 기본 정보를 포함합니다.

* <Cmdname sid="start_time" className="uitext" />: <b>페이지 로드</b> 이벤트의 시작 시간

* <Cmdname sid="elapsed" className="uitext" />: <b>페이지 로드</b> 이벤트 완료까지의 소요 시간

* <Cmdname sid="page_url" className="uitext" />: 최종 사용자가 접속한 페이지 URL 경로

* <Cmdname sid="browser" className="uitext" />: 최종 사용자가 사용한 브라우저 종류

* <Cmdname sid="device" className="uitext" />: 최종 사용자가 사용한 기기 종류

* <Cmdname sid="os" className="uitext" />: 최종 사용자가 사용한 기기의 운영체제 종류

* <Cmdname sid="client_ip" className="uitext" />: 최종 사용자가 사용한 기기의 IP 주소

테이블에 다른 정보 컬럼을 추가하거나, 추가한 컬럼을 숨길 수도 있습니다. <Cmdname sid="page" className="uitext" /> <Cmdname sid="list" className="uitext" /> 위젯의 오른쪽 위에 ![컬럼 아이콘](/img/ico-column.svg) 버튼을 선택하세요.

:::note

## 페이지 로드 이벤트 상세 정보 {#pageload-details}

<Cmdname sid="page" className="uitext" /> <Cmdname sid="list" className="uitext" />에서 각 항목의 가장 왼쪽에 <LinkImage img="ico-detail-view.svg" className="ico" desc="상세 보기 아이콘" /> 버튼을 선택하면 <Cmdname sid="page_load" className="uitext" /> <Cmdname sid="detail" className="uitext" /> 창이 나타납니다. <b>페이지 로드</b>의 성능을 분석하기 위해 다양한 지표를 제공하고, 성능에 영향을 주는 요소를 파악할 수 있습니다.

<ImgLang img='rum-pageload-event-02.png' desc='페이지 로드 상세' />

### ![](/img/number-01.png) URL 목록 {#url-list}

<Cmdname sid="page" className="uitext" /> <Cmdname sid="list" className="uitext" />의 항목을 URL 기준으로 확인할 수 있습니다. <Cmdname sid="url_list" className="uitext" />의 정렬 기준은 경과 시간입니다. URL 항목을 선택하면 <b>페이지 로드</b> 이벤트에 대한 상세 정보를 <LinkImage img="number-02.png" className="ico" desc="숫자 2" /> 오른쪽 화면에서 조회할 수 있습니다.

* 각 항목의 파랑색 막대 그래프는 <b>페이지 로드</b> 경과 시간입니다.
* 각 항목의 막대 그래프 오른쪽의 시간은 <b>페이지 로드</b> 시작 시간입니다.

### ![](/img/number-02.png) 개요(페이지 로드 이벤트 요약) {#pageload-events-summary}

<Cmdname sid="summary" className="uitext" /> 탭에서는 <b>페이지 로드</b> 과정 중 구간별 소요된 시간을 그래프 차트로 제공합니다. 이를 통해 <b>페이지 로드</b> 과정 중 느린 구간을 파악할 수 있습니다.

* <Cmdname sid="page" className="uitext" /> <Cmdname sid="info" className="uitext" />: 최종 사용자의 접속 환경, 접속 페이지 정보, <Cmdname sid="user_id" className="uitext" />, <Cmdname sid="session_id" className="uitext" /> 정보를 제공합니다.

* <Cmdname sid="page_load_timing" className="uitext" /> <Cmdname sid="info" className="uitext" /> : <b>페이지 로드</b> 시 구간별 소요 시간을 확인할 수 있습니다.

  * ***redirect***: 브라우저가 리디렉션에 소요한 시간
  
  * ***cache***: 브라우저가 캐시된 리소스를 검색하는데 소요한 시간
  
  * ***dns***: 브라우저가 도메인을 조회하는데 소요한 시간
  
  * ***wait***: 브라우저가 <b>페이지 로드</b>에 필요한 데이터에 대한 응답을 기다리는데 소요한 시간
  
  * ***download***: 브라우저가 서버로부터 리소스를 다운로드하는데 소요한 시간
  
  * ***render***: 서버로부터 다운로드한 리소스를 화면에 렌더링하고 <b>페이지 로드</b> 이벤트를 완료하는데 걸린 시간

### 리소스 목록 {#resource-list}

브라우저가 서버로부터 다운로드하는 리소스는 <b>페이지 로드</b> 성능에 큰 영향을 줄 수 있습니다. <Cmdname sid="resource" className="uitext" /> <Cmdname sid="list" className="uitext" />에서는 로딩 속도가 느리거나 파일 사이즈가 큰 리소스를 빠르게 파악할 수 있습니다.

<ImgLang img='rum-pageload-event-03.png' desc='리소스 목록' />

각 리소스의 시작 시간을 기준으로 타임라인 차트를 제공해 로딩 속도가 느린 리소스를 파악할 수 있습니다. 각 리소스의 파일 크기를 같이 참조해 성능에 영향을 끼치는 부분을 개선할 수 있습니다. 각 리소스를 클릭하면 <Cmdname sid="resource" className="uitext" /> <Cmdname sid="detail" className="uitext" /> 창을 통해 상세 시간 정보를 확인할 수 있습니다.

<ImgLang img='rum-pageload-event-04.png' desc='리소스 상세' />

### 브라우저 에러 확인하기

브라우저 에러 데이터를 포함하고 있는 페이지 로드 이벤트의 경우 빨간색으로 표시됩니다. <Cmdname sid="resource" className="uitext" /> <Cmdname sid="list" className="uitext" />에서 에러가 발생한 스텝을 확인할 수 있습니다.

<ImgLang img='browser-pageload-details-resource-error.png' desc='리소스 목록 - 에러 데이터' />

1. <Cmdname sid="resource" className="uitext" /> <Cmdname sid="list" className="uitext" />에서 에러가 발생한 스텝을 선택하면 <Cmdname sid="exception_stack" className="uitext" /> 창이 나타납니다.

    <ImgLang img='browser-pageload-details-resource-error-stack.png' desc='에러 스택' />

1. ***Upload*** 버튼을 선택하세요.

1. 해당 코드 스택에 대한 소스 맵 파일을 선택하세요.

난독화된 스택 정보와 자바스크립트 소스맵 정보를 비교하여 실제 에러가 발생한 코드를 확인할 수 있습니다.
