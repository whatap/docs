<Cmdname sid="side_pageLoadStatistics" className="uitext" /> 메뉴는 웹 사이트의 페이지별 로드 성능 정보를 제공합니다. 이 메뉴을 통해 각 페이지가 로드되는 과정을 상세하게 추적할 수 있으며, 성능이 저하된 페이지를 식별하고 개선된 페이지의 성과를 분석할 수 있습니다. 또한 웹 애플리케이션의 사용자 경험을 개선하기 위한 유용한 정보를 얻을 수 있습니다.

- 페이지 URL 경로별 로드 성능 통계를 제공합니다.

- 평균값, 최댓값, 최솟값, 백분위수 기반 통계를 확인할 수 있습니다.

- 페이지 로드 과정의 세부 시간에 대한 필터를 통해 성능을 집중 분석할 수 있습니다.

- Navigation Timing API를 통해 페이지 로드의 각 단계 별 시간을 수집하여 통계를 제공합니다.

:::note

<Cmdname sid="side_pageLoadStatistics" className="uitext" /> 메뉴에서 수집하는 데이터에 대한 자세한 내용은 [다음 문서](collect-data#pageloadtime)를 참조하세요.

:::

## 기본 사용 안내

페이지의 URL 경로를 기준으로 페이지 로드 시간 및 DOM 로드 시간, 네트워크 지연 시간, 리소스 로드 시간 등 다양한 성능 지표를 제공합니다. 이를 통해 페이지 성능 최적화 여부를 확인할 수 있습니다

<ImgLang img="browser-pageload-statistics.png" desc="페이지 로드 통계" />

1. <Cmdname sid="time" className="uitext" />에서 성능 지표를 조회할 날짜와 시간을 설정하세요. 

1. <Cmdname sid="TTL08276" className="uitext" />에서 원하는 기준을 선택하세요.

1. 화면 오른쪽 상단의 ![검색 아이콘](/img/ic-search-on-press.svg) 버튼을 선택하세요.

조회한 시간 범위의 성능 지표를 각 섹션에서 그래프 차트와 목록을 통해서 확인할 수 있습니다. 각 섹션의 그래프 차트로 마우스를 오버하면 해당 시간대의 지표 정보를 확인할 수 있습니다.

:::note

<Cmdname sid="TTL08276" className="uitext" />에 대한 자세한 내용은 다음 문서를 참조하세요.

### 조회 시간 설정하기

<Cmdname sid="time" className="uitext" />에서 성능 지표를 조회할 날짜와 시간을 설정하세요.

* 가장 오른쪽의 녹색 버튼을 선택하면 다양한 시간 범위를 선택할 수 있습니다. 

  <ImgLang img="browser-statistics-set-time-select-options.png" className="width-600" desc="시간 설정" />

* ![왼쪽 방향 아이콘](/img/left-arrow.svg) 또는 ![오른쪽 방향 아이콘](/img/right-arrow.svg) 버튼을 선택해 시간을 이동할 수 있습니다. 이동 간격은 오른쪽에 설정된 시간 간격입니다.

* 원하는 날짜와 시간을 직접 선택하려면 날짜 및 시간 영역을 클릭하세요.

  <ImgLang img="browser-statistics-set-custom-time.png" desc="시간 설정" />

:::note

* 최대 선택할 수 있는 시간은 3일이며, 최소 선택할 수 있는 시간은 5분입니다.

* 2시간 이하 범위를 선택하면 5분 단위로 데이터를 조회하고, 2시간을 초과하면 1시간 단위로 데이터를 조회합니다.

:::

### 통계 기준 선택하기

<Cmdname sid="TTL08276" className="uitext" />은 사용자가 분석하려는 통계 지표를 다음 선택한 기준으로 확인할 수 있는 옵션입니다.

* <Cmdname sid="merge_avg" className="uitext" />: 각 페이지의 평균 로드 시간을 계산하여 통계를 제공합니다.

* <Cmdname sid="merge_max" className="uitext" />: 각 페이지의 최대 로드 시간을 기준으로 통계를 제공합니다. 즉, 페이지 로드 시간이 가장 비효율적인 페이지 URL을 파악할 수 있습니다.

* <Cmdname sid="merge_min" className="uitext" />: 각 페이지의 최소 로드 시간을 기준으로 통계를 제공합니다. 이를 통해 가장 최적의 조건에서 로드된 페이지 URL을 확인할 수 있습니다. 또는 성능 최적화 목표를 설정할 때 유용합니다.

* <Cmdname sid="ITM08291" className="uitext" />: 각 페이지 백분위수 기준 로드 시간에 대한 통계를 제공합니다.

  - <Cmdname sid="TTL08288" className="uitext" replace="95" />: 가장 느린 5%의 페이지 로드 시간으로, 페이지 로드 시간이 가장 느렸던 상위 5% 사용자의 경험을 의미합니다.

  - <Cmdname sid="TTL08288" className="uitext" replace="90" />: 가장 느린 10% 사용자의 페이지 로드 시간으로, 주로 느린 사용자 그룹의 성능 문제를 파악할 때 유용합니다.

  - <Cmdname sid="TTL08288" className="uitext" replace="75" />: 페이지 로드 시간이 하위 75%보다 느린 상위 25% 사용자의 로드 시간을 보여줍니다.

  - <Cmdname sid="TTL08288" className="uitext" replace="50" />: 중간값으로, 전체 사용자 중 절반은 해당 시간보다 빠르게 페이지를 로드하고, 절반은 더 느리게 로드한 것을 의미합니다.

:::tip

* <Cmdname sid="merge_avg" className="uitext" />은 전체적인 성능의 흐름을 파악하는 데 유용하지만, 성능 저하가 간헐적으로 발생하는 경우를 파악하기 어려울 수 있습니다. 이때 <Cmdname sid="merge_max" className="uitext" />과 <Cmdname sid="ITM08291" className="uitext" /> 옵션을 함께 활용하면 성능 저하가 언제 발생했는지, 그 심각성은 어느 정도인지 확인할 수 있습니다.

* <Cmdname sid="merge_min" className="uitext" />은 성능 최적화 목표를 설정하거나 특정 최적화가 얼마나 효과적인지 평가할 때 참고할 수 있습니다. 하지만 이 값만으로는 성능의 일관성을 평가하기 어렵기 때문에 <Cmdname sid="merge_avg" className="uitext" />이나 <Cmdname sid="ITM08291" className="uitext" /> 옵션과 함께 통계 정보를 분석하는 것이 좋습니다.

* <Cmdname sid="ITM08291" className="uitext" /> 옵션은 단순한 평균보다 더 세밀하게 성능 데이터를 분석하는 데 적합하며 사용자 경험을 개선하는 데 중요한 데이터를 제공합니다. 이 옵션을 활용하면 성능 저하가 발생하는 사용자의 비율을 파악하고 최적화 대상 페이지를 효과적으로 선택할 수 있습니다.

:::

### 조회 결과 다운로드하기

  :::

* <Cmdname sid="TTL08446" className="uitext" />

  페이지 URL 경로(Path)를 기준으로 다양한 성능 지표를 한눈에 확인할 수 있는 테이블 목록입니다. 정규 표현식으로 그룹화하지 않은 URL path를 기준으로 목록을 제공합니다. 목록의 가장 왼쪽의 ![상세 아이콘](/img/ico-detail-view.svg) <Cmdname sid="detail" className="uitext" /> 버튼을 선택하면 페이지 URL path 기준 <Cmdname sid="TTL08448" className="uitext" /> 화면을 확인할 수 있습니다. 컬럼 항목에 대한 자세한 내용은 [다음 문서](#columninfo)를 참조하세요.

  :::note

  * **데이터 수집 제한**

    - 5분마다 최대 5,000개의 페이지 URL path 목록을 생성하여 서버로 전송합니다.
    
    - 5분 동안 고유한 URL path의 수가 5,000개를 초과하면 추가 데이터는 무시됩니다.

  * **데이터 정확성**

    **Navigation Timing API**에서 일부 세부 시간 정보가 0으로 수집될 수 있습니다. 이로 인해 특정 성능 지표가 표시되지 않을 수 있습니다.

  * **차트 간격**

    조회하는 데이터의 시간 단위에 따라 차트 간격이 결정됩니다.

    - 1시간 데이터로 조회하면 차트 간격은 1시간으로 표시됩니다.

    - 5분 데이터로 조회하면 차트 간격은 5분으로 표시됩니다.

  :::

## 조회 결과 필터링하기

조회한 통계 데이터를 <Cmdname sid="page_group" className="uitext" /> 또는 <Cmdname sid="TTL08292" className="uitext" />를 기준으로 필터링하여 원하는 내용을 집중 분석할 수 있습니다.

### 페이지 그룹 필터

특정 URL과 관련한 페이지를 선택해 성능 데이터를 조회할 수 있습니다. <Cmdname sid="page_group" className="uitext" /> 옵션에서 원하는 페이지 그룹 항목을 선택하세요.

<ImgLang img="browser-statistics-page-group-filter.png" desc="페이지 그룹" />

:::note

페이지 그룹은 최대 3개까지 선택할 수 있습니다.

:::

### 성능 지표 필터

성능 지표는 페이지 로드 시간, 리다이렉트 시간, 다운로드 시간 등을 의미합니다. 다양한 성능 지표에 임계값을 적용하여 조회할 수 있습니다. <Cmdname sid="TTL08292" className="uitext" /> 옵션에서 원하는 지표 항목을 선택한 다음 임계값과 연산자 조건을 적용하세요.

<ImgLang img="browser-statistics-perf-metrics-filter.png" desc="성능 지표" />

* 연산자 종류: `>`(보다 큼), `>=`(보다 크거나 같음), `<`(보다 작음), `<=`(보다 작거나 같음), `==`(같음)

* 임곗값으로 양수, 음수, 0을 입력할 수 있지만, 선택한 성능 지표에 따라 음수 입력 여부는 다를 수 있습니다.

* 한 개의 성능 지표에 대해 필터를 적용하고 조회할 수 있습니다.

:::note

선택할 수 있는 성능 지표에 대한 자세한 내용은 [다음 문서](#columninfo)를 참조하세요.

:::

## 컬럼 설정하기

<Cmdname sid="TTL08445" className="uitext" />에서 테이블 헤더 컬럼을 감추거나 원하는 항목을 추가할 수 있습니다. 컬럼 순서를 변경할 수도 있습니다. ![컬럼 아이콘](/img/ic-column.svg) <Cmdname sid="BTN06441" className="uitext" /> 버튼을 선택하세요.

<ImgLang img="browser-statistics-setting-column.png" desc="컬럼 설정" />

:::note

왼쪽 목록을 오른차순 또는 내림차순으로 정렬하려면 ![정렬 아이콘](/img/ic-sort-up.svg) / ![정렬 아이콘](/img/ic-sort-down.svg) 버튼을 선택하세요.

:::

### 컬럼 추가하기

1. 왼쪽 목록에서 추가할 컬럼 항목을 선택하세요.

1. 선택한 컬럼 항목이 오른쪽 목록에 추가된 것을 확인하세요.

1. 원하는 컬럼 항목을 모두 추가했으면 <Cmdname sid="confirm" className="uitext" /> 버튼을 선택하세요.

:::note

* 모든 컬럼 항목을 선택하려면 왼쪽 목록 상단의 <Cmdname sid="select_all" className="uitext" />을 클릭하세요.

* 컬럼 항목에 대한 자세한 내용은 [다음 문서](#columninfo)를 참조하세요.

:::

### 컬럼 삭제하기

1. 오른쪽 목록에서 삭제할 컬럼 항목의 오른쪽에 ![삭제 아이콘](/img/ico-remove.svg) 버튼을 선택하세요.

1. 왼쪽 목록에서 삭제한 컬럼 항목의 체크 박스가 해제된 것을 확인하세요.

1. 원하는 컬럼 항목을 모두 삭제했으면 <Cmdname sid="confirm" className="uitext" /> 버튼을 선택하세요.

### 컬럼 순서 변경하기

다음 제공하는 기능을 통해 컬럼의 순서를 변경하거나 원하는 항목을 고정할 수 있습니다.

* 오른쪽 목록에서 순서를 변경할 항목을 드래그해서 원하는 위치로 이동해 컬럼의 순서를 변경할 수 있습니다. 

* 컬럼 항목 오른쪽의 ![핀 아이콘](/img/ic-pin.svg) 버튼을 선택하세요. 선택한 컬럼 항목은 최상단에 고정됩니다. 고정된 컬럼 항목은 테이블 목록에서 가장 왼쪽에 고정됩니다.

모든 설정을 완료한 후에는 <Cmdname sid="confirm" className="uitext" /> 버튼을 선택하세요.

### 컬럼 설정 초기화하기

변경 사항을 모두 취소하고 초기화하려면 ![초기화 아이콘](/img/ico-reset.svg) <Cmdname sid="initialized" className="uitext" /> 버튼을 선택하세요. 초기화 설정을 반영하려면 <Cmdname sid="confirm" className="uitext" /> 버튼을 선택하세요.
