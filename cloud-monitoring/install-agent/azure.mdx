**WhaTap Azure Monitor**는 API 연동을 통해 주기적으로 메트릭을 수집합니다. 사용자는 모니터링할 서비스의 Azure Monitor 서비스를 선택하고, 5분 단위로 원하는 통계(Stat) 항목을 지정해 데이터를 가져올 수 있습니다.
모니터링 시작하면 모든 리전의 자원을 자동으로 검색하여, 한 번의 클릭으로 모니터링을 시작할 수 있습니다.
또한, WhaTap은 안전하고 편리한 연동을 위해 Azure Registered App 방식을 지원합니다.

### Azure Registered App 추가 {#add-azure-registered-app}

WhaTap 모니터링을 위해 Monitor Reader, Reader, API Management Service Reader 권한이 필요합니다. Azure Portal에서 **Subscription** &gt; **액세스 제어**(IAM)에서 **추가** 버튼을 클릭해 권한을 추가할 수 있습니다.

1. WhaTap 모니터링용 [Registered App](https://portal.azure.com/#view/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/~/RegisteredApps)을 생성하세요.

    * Registered App은 Azure AD(Entra ID)에 등록된 애플리케이션으로, WhaTap이 Azure API에 접근할 수 있도록 인증 정보를 제공합니다.

    ![Azure Active Directory](/img/registeredapps.png)

1. Secret을 생성하고 애플리케이션 ID(Application Id), 테넌트 ID(Tenant Id)와 시크릿(Azure secret key)을 WhaTap에 입력하세요. Secret은 비밀번호처럼 취급되므로 외부에 노출되지 않도록 주의하세요.

### 인증 정보 입력 {#authentication}

1. 와탭 모니터링 서비스에 생성한 프로젝트의 <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="agentInstall" className="uitext" />로 이동합니다.

1. Azure 인증 정보인 **애플리케이션 ID**(Application ID), **테넌트 ID**(Tenant ID)와 **Azure 시크릿 키**(Azure secret key)를  입력하세요.

### 리소스 그룹, 메트릭스, 스탯 선택 {#region-metric-stats}

1. 와탭 모니터링 서비스에 생성한 프로젝트의 <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="agentInstall" className="uitext" />로 이동합니다.

1. 활성화된 서비스의 오른쪽 ![수정 아이콘](/img/ico-edit.svg)**수정** 버튼을 클릭하면 수집할 **리소스 그룹**, **메트릭스**, **스탯**을 선택할 수 있습니다. 

1. **저장** 버튼을 클릭합니다.

## 데이터 수집 비활성화 {#disable-collection}

1. 와탭 모니터링 서비스에 생성한 프로젝트의 <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="agentInstall" className="uitext" />로 이동합니다.

1. 수집 토글 ![아이콘](/img/ic-toggle-off.svg) 비활성화 시 데이터 수집이 중지됩니다.

## 서비스 초기화 {#service-reset}

{/* <ImgLang img='cloud-deact.png' desc='비활성화 및 서비스 초기화 sc' /> */}

1. 와탭 모니터링 서비스에 생성한 프로젝트의 <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="agentInstall" className="uitext" />로 이동합니다.

1.  **역할(Role) 변경** 버튼을 클릭해 **에이전트 설치** 화면으로 이동한 후, **서비스 초기화** 선택 시 메트릭을 수집 중인 모든 서비스가 비활성화됩니다.

    * 완전한 비용 발생 방지를 위해서는 다음을 제거해야 합니다.  

        * Azure Portal Subscript에서 모니터링 리더 Role을 제거하세요. 

        * Azure Registered App을 제거하세요.
