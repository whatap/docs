**WhaTap Oracle Cloud Monitor**는 API 연동을 통해 메트릭을 주기적으로 수집합니다. 수집을 원하는 서비스에 해당하는 Oracle Cloud Monitor 서비스를 5분 간격으로 원하는 스탯을 선택해 수집합니다. 모니터링 시작 시 모든 리전의 사용자 자원을 검색하여 원클릭으로 모니터링을 시작할 수 있습니다. Oracle Cloud API 등록 시 필요한 ssh 공개 키를 자동 생성하여 안전하고 편리하게 사용할 수 있습니다. 


## 모니터링 에이전트 설치 {#monitoring-agent-install}

WhaTap Oracle Cloud Monitor 설치하려면 아래 단계를 순서대로 진행합니다.

[Step 1. Oracle Cloud 인증 정보를 입력하세요.](#authentication)  
[Step 2. Oracle Cloud Monitoring 서비스 및 수집 주기를 선택하세요.](#namespace-and-collecting-period)  
[Step 3. 리전, 메트릭, 스탯 등을 선택하세요.](#region-metric-stats)

:::note

설치 후 사용량에 따라 와탭 요금과는 별개로 클라우드에서 추가 비용이 발생할 수 있습니다. 설치 전 사용 중인 클라우드 서비스의 가격 정책을 [다음 문서](https://www.oracle.com/kr/cloud/price-list/)에서 확인하세요.

:::

### 인증 정보 입력 {#authentication}

1. 와탭 모니터링 서비스에 생성한 프로젝트의 <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="agentInstall" className="uitext" />에서 **RSA Public Key 생성** 버튼을 클릭해, RSA Public Key 생성 후 메모장에 생성한 RSA Publkic Key를 복사합니다. 

    :::note

    이후 **설치 안내** 탭에서 **RSA Public Key 생성** 버튼을 클릭하면, 생성된 KEY를 언제든지 확인할 수 있습니다.
    
    :::

    <ImgLang img='oci-install.png' desc='OCI Monitor' />

1. Oracle Cloud 콘솔의 Identity & Security로 이동하세요. 

1. Identity & Security 메뉴 하단의 Groups를 선택해 모니터링에 사용할 **Group**을 신규 생성하거나 선택하세요.

1. Identity & Security 메뉴 하단의 Policies를 선택해 모니터링용 **Policy**를 생성하세요.

    * Create Policy를 클릭하여, 이름, Descrition 및 Compartment를 입력하세요.
    * Policy Builder에서 Show Manual Editor를 선택해, 아래 권한을 입력 후 Create를 클릭하세요.
        > * Allow group `{그룹}` to read metrics in compartment `{컴파트먼트}`
        > * Allow group `{그룹}` to inspect metrics in compartment `{컴파트먼트}`

1. 오른쪽 상단의 Identity &gt; My profile를 선택하세요. 

    * 왼쪽 하단의 API Keys를 클릭하세요.
    * Add API Key 버튼을 클릭하세요.
    * 라디오 버튼의 Paste Public Key를 선택하여 2단계에서 받은 **RSA Public Key**를 입력하세요.
    * 추가 완료 시 **User**, **Tenancy** 정보를 화면에서 확인할 수 있습니다. 이 정보를 메모장에 복사하세요.

1.  Oracle Cloud 콘솔의 Identity & Security 하단 Compartments를 선택해 모니터링할 Compartment의 **OCID**를 확인 및 메모장에 복사하세요.

1.  와탭 모니터링 서비스에 생성한 프로젝트의 <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="agentInstall" className="uitext" />로 돌아와 5.에서 확인한 **Tenancy**와 **User** 및 6.에서 확인한 **CompartmentID**를 입력하세요.

1. **확인** 버튼을 선택하세요.

## 데이터 수집 비활성화 {#disable-collection}

1. 와탭 모니터링 서비스에 생성한 프로젝트의 <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="agentInstall" className="uitext" />로 이동합니다.

1. 수집 토글 ![아이콘](/img/ic-toggle-off.svg) 비활성화 시 데이터 수집이 중지됩니다.

## 서비스 초기화 {#service-reset}

{/* <ImgLang img='cloud-deact.png' desc='비활성화 및 서비스 초기화 sc' /> */}

1. 와탭 모니터링 서비스에 생성한 프로젝트의 <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="agentInstall" className="uitext" />로 이동합니다.

1.  **역할(Role) 변경** 버튼을 클릭해 **에이전트 설치** 화면으로 이동한 후, **서비스 초기화** 선택 시 메트릭을 수집 중인 모든 서비스가 비활성화됩니다.

    * 완전한 비용 발생 방지를 위해서는 다음을 제거해야 합니다.  

        * Oracle Cloud 콘솔 Identity & Security 메뉴에서 도메인과 사용자 선택 후 API Keys의 Fingerprint를 제거하세요.
