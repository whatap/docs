WhaTap 클라우드 모니터링은 AWS, Azure, Oracle Cloud, Naver Cloud 등 다양한 클라우드 서비스의 메트릭을 한 화면에서 통합 관리할 수 있습니다. 복잡한 설정 없이 API 연동만으로 손쉽게 자원 사용 현황을 실시간으로 파악할 수 있습니다. 

<ImgLang img='cloud-monitoring-intro.png' desc='Cloud Monitoring intro' />

## 목적

클라우드 환경에서 수많은 리소스가 동적으로 생성되고 사라지기 때문에, 서비스 상태를 실시간으로 파악하지 않으면 장애 대응이 늦어지고 리소스가 낭비될 수 있습니다. 와탭 클라우드 모니터링은 API를 통해 메트릭을 수집하고, 다양한 리소스 상태를 한눈에 확인할 수 있는 대시보드를 제공함으로써 다음과 같은 목적에 활용됩니다.

* **장애 조기 감지 및 신속한 대응**

* **리소스 운영 효율화**

* **운영 가시성 확보**

* **성능 개선과 최적화**

## 특징

* **통합 대시보드 제공**

    여러 클라우드 플랫폼의 메트릭을 하나의 화면에서 비교하고 분석할 수 있습니다.

* **API 기반 실시간 수집**

    각 클라우드 플랫폼의 API를 통해 CPU, 메모리, 네트워크 등 주요 자원 사용량을 주기적으로 수집합니다.

* **리전 전체 자원 자동 탐색**

    모니터링 시작 시 모든 리전의 사용자 자원을 자동으로 검색하고, 원클릭으로 모니터링을 시작할 수 있습니다.

* **간편한 시작**

    별도의 복잡한 설정 없이, 계정 연결만으로 모니터링을 빠르게 시작할 수 있습니다.

## 지원하는 클라우드

* **Amazon CloudWatch**

Amazon CloudWatch API를 통해 주기적으로 메트릭을 검색하고 저장합니다.

* **Amazon ECS**

META API 및 cgroup 디렉토리를 통해 도커 컨테이너별 자원 사용량을 실시간으로 수집합니다.

* **Microsoft Azure**

Azure Monitor API를 통해 주기적으로 메트릭을 검색하고 저장합니다.

* **Oracle Cloud**

Oracle Cloud Monitor API를 통해 주기적으로 메트릭을 검색하고 저장합니다.

* **Naver Cloud**

Naver Cloud Monitoring API를 통해 주기적으로 메트릭을 검색하고 저장합니다.
