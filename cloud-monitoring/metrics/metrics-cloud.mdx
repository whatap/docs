**지표**는 클라우드나 온프레미스 환경의 리소스에서 수집된 성능 데이터를 시계열 형태로 표현한 값입니다.  
CPU 사용률, 메모리 점유율, 요청 수와 같은 수치는 메트릭스로 수집되어 시스템 상태를 정량적으로 파악하고, 대시보드나 알림 설정 등에 활용됩니다.

와탭은 다양한 클라우드 서비스로부터 메트릭스를 자동으로 수집하고, 이를 규칙에 따라 카테고리별로 관리합니다.

## 클라우드별 메트릭스 수집 규칙

클라우드별 메트릭스 네임스페이스 표기 규칙과 수집 기준입니다.

### Amazon CloudWatch

* 네임스페이스는 와탭 메트릭스 **카테고리**로 수집됩니다. 표기 형식은 Amazon CloudWatch 네임스페이스를 **소문자**로 변경 후 '/'를 '**_**'로 변경합니다.

  > 예, AWS/EC2 → **aws_ec2**

* 지표는 와탭 **메트릭스**로 수집됩니다. 표기 형식은 'CloudWatchMetric.Stat' 입니다.

  > 예, **CPUUtilization.Average**

* 수집되는 Amazon CloudWatch 지표는 [다음 문서](https://docs.aws.amazon.com/ko_kr/AmazonCloudWatch/latest/monitoring/aws-services-cloudwatch-metrics.html)를 참조하세요.

### Amazon ECS

* 와탭 메트릭스 **카테고리**의 표기는 접두사(**ecs_**)를 활용합니다.

* 지표는 와탭 **메트릭스**로 수집됩니다. [다음 문서](metrics-ecs)를 참조하세요.

### Azure Monitor

* 네임스페이스는 와탭 메트릭스 **카테고리**로 수집됩니다. 표기 형식은 Auzure Monitor 네임스페이스를 **소문자**로 변경 후 '/'를 '**_**'로 변경합니다.
  > 예, **azure**

* 지표는 와탭 **메트릭스**로 수집됩니다. 표기 형식은 'AzureMonitorMetric.Stat' 입니다.

* 수집되는 Auzure Monitor 지표는 [다음 문서](https://learn.microsoft.com/ko-kr/azure/azure-monitor/essentials/metrics-supported)를 참조하세요.


### Naver Cloud

* 네임스페이스는 와탭 메트릭스 **카테고리**로 수집됩니다. 표기 형식은 Naver Cloud 네임스페이스를 **소문자**로 변경 후 '/'를 '**_**'로 변경합니다.

  > 예, **ncloud**

* 지표는 와탭 **메트릭스**로 수집됩니다. 표기 형식은 'NaverCloudMetric.Stat' 입니다.

* 수집되는 Naver Cloud 지표는 [다음 문서](https://api.ncloud-docs.com/docs/management-cloudinsight-productinfo)를 참조하세요.

### Oracle Cloud

* 네임스페이스는 와탭 메트릭스 **카테고리**로 수집됩니다. 표기 형식은 Oracle Cloud 네임스페이스를 **소문자**로 변경 후 '/'를 '**_**'로 변경합니다.

  > 예, **oci**

* 지표는 와탭 **메트릭스**로 수집됩니다. 표기 형식은 'OracleCloudMetric.Stat' 입니다.

* 수집되는 Oracle Cloud 지표는 [다음 문서](https://docs.oracle.com/en-us/iaas/Content/Monitoring/Concepts/monitoringoverview.htm#SupportedServices)를 참조하세요.
