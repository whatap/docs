비즈니스 대시보드는 애플리케이션의 비즈니스 단위(App Context)별 성능을 모니터링할 수 있는 대시보드입니다.
URL 패턴이나 경로를 기반으로 정의된 각 비즈니스 항목에 대해 실시간 및 과거의 성능 데이터를 조회하고 분석할 수 있습니다.
TPS, 응답시간, 에러율 등의 주요 성능 지표를 비즈니스 단위로 확인함으로써, 특정 업무나 서비스의 성능 상태를 빠르게 파악할 수 있습니다.

## 화면 구성

### 시간 선택

화면 상단의 시간 선택자를 통해 조회할 시간 범위를 설정할 수 있습니다.

- **실시간 모드**: 5초 주기로 자동 갱신되며 최근 10분의 데이터를 표시합니다.
- **과거 조회 모드**: 최대 15일까지 과거 데이터를 조회할 수 있습니다.
- **데이터 집계 단위**:
    - 3시간 미만: Raw 데이터 사용
    - 3시간 ~ 3일: 5분 통계 데이터 사용
    - 3일 이상: 1시간 통계 데이터 사용

### 에이전트 선택

시간 선택자 아래의 에이전트 선택기를 통해 특정 에이전트나 에이전트 그룹의 데이터만 필터링하여 볼 수 있습니다.

### 뷰 모드 변경

뷰 모드 변경 버튼 및 검색 영역

화면 좌측 상단의 버튼을 통해 두 가지 뷰 모드를 전환할 수 있습니다:

- **테이블 뷰**: 전체 비즈니스 목록을 테이블 형태로 표시합니다.
- **카드 뷰**: 개별 비즈니스를 카드 형태로 표시합니다. 카드 뷰에서는 열 개수(1~4열)를 선택할 수 있습니다.

### 업무명 검색

검색창에서 업무명을 입력하여 특정 비즈니스를 빠르게 찾을 수 있습니다.

### CSV 다운로드

우측 상단의 CSV 다운로드 버튼![CSV 아이콘](/img/ic-csv-download-ms.svg)을 클릭하여 현재 화면에 표시된 모든 비즈니스의 성능 데이터를 CSV 파일로 다운로드할 수 있습니다.

### 성능 지표

각 비즈니스별로 다음과 같은 성능 지표를 확인할 수 있습니다:

- **TPS**: 초당 트랜잭션 처리량
- **트랜잭션 수**: 조회 기간 동안의 총 트랜잭션 수
- **트랜잭션/min**: 분당 트랜잭션 수
- **응답시간**: 평균 및 최대 응답시간 (차트와 함께 표시)
- **에러 수**: 조회 기간 동안의 총 에러 수
- **에러율(%)**: 전체 트랜잭션 대비 에러 비율 (차트와 함께 표시)
- **에러/min**: 분당 에러 수
- **액티브 트랜잭션**: 현재 진행 중인 트랜잭션 상태 (스피드미터로 표시)

## 에이전트 설정

비즈니스 대시보드를 사용하려면 에이전트에서 관련 설정이 필요합니다.

### 지원 에이전트 버전

- **Java**: 2.2.40 이상
- **PHP**: 2.11.0 이상
- **Python**: 1.8.5 이상
- **.NET**: 2.4.3 이상

### 필수 설정

```
# 기능 전체 활성화 여부
app_context_enabled=true
```

### 선택 설정

```
# 파서 선택 (default/prefix/match)
# default: 첫 번째 경로 세그먼트 추출
# prefix: 접두사 매칭 파서 (가장 유연함)
# match: 정확한 매칭 파서 (완전 일치만)
app_context_parser=default

# 파서의 내부 캐시와 설정을 강제로 재설정하는 카운터
app_context_parser_reset=0

# URL에서 추출할 경로 세그먼트의 깊이 지정 (default 파서에서 1 또는 2로 사용)
app_context_path_depth=1

# 모니터링할 경로와 해당 이름을 정의하는 매핑 설정 (쉼표로 구분하여 사용)
app_context_path_set=name1@url1,name2@url2,url3,name4@url4
```

에이전트 설정을 완료한 후에는 애플리케이션을 재시작해야 설정이 적용됩니다.

## 참고

- 비즈니스 대시보드에 데이터가 표시되지 않는 경우, 에이전트 설정을 확인하세요.
- 에이전트 버전이 지원 버전보다 낮은 경우, 에이전트 업데이트가 필요합니다.
- 비즈니스 컨텍스트는 URL 패턴을 기반으로 자동 또는 수동으로 정의할 수 있습니다.
