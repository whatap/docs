컬럼 정보에 대한 보다 자세한 내용은 [다음 링크](https://learn.microsoft.com/en-us/sql/sql-server/?view=sql-server-ver16)를 참조하세요.

<Tabs groupId="db-columns">

<TabItem value="activesession" label="Active session">

| 항목 | 설명 |
| -- | -- |
| `application` | 세션을 생성한 애플리케이션의 이름입니다. |
| `blocking_session_id` | 현재 세션을 차단하고 있는 세션의 ID입니다. |
| `client` | 세션을 생성한 클라이언트의 이름 또는 IP 주소입니다. |
| `command` | 세션이 실행 중인 SQL 명령어입니다. (SELECT, INSERT, UPDATE, DELETE) |
| `completion_time` | SQL 작업이 완료될 것으로 예상되는 시간입니다.|
| `cpu(%)` | 해당 세션이 사용하는 CPU 비율입니다.|
| `cpu_time`| 세션에서 사용된 CPU 시간(밀리초 단위)입니다. |
| `cpu_time(sigma)` | 모든 세션에서 사용된 총 CPU 시간입니다. |
| `db` | 세션이 연결된 데이터베이스의 이름입니다.|
| `elapsed_time`| 세션이 시작된 이후 경과된 시간(밀리초 단위)입니다. |
| `granted_query_memory` | 쿼리 실행을 위해 할당된 메모리 양(페이지 단위)입니다. |
| `host`| 세션을 생성한 클라이언트 호스트의 이름입니다. |
| `isolation` | 현재 세션의 트랜잭션 격리 수준입니다. |
| `last_request_start_time` | 세션의 마지막 요청이 시작된 시간입니다. |
| `last_wait_type` | 세션이 마지막으로 기다린 대기 유형입니다. |
| `login_time` | 세션이 SQL Server에 로그인한 시간입니다.|
| `logical_reads` | 세션에서 발생한 논리적 읽기 횟수입니다. |
| `logical_reads(sigma)`| 모든 세션에서 발생한 총 논리적 읽기 횟수입니다. |
| `memory_usage`| 세션이 사용하는 메모리 양입니다.|
| `object` | 현재 실행 중인 SQL 문이 참조하는 데이터베이스 객체의 이름입니다. |
| `objectid`| 세션에서 실행 중인 객체의 ID입니다. |
| `percent_complete`| 현재 실행 중인 명령의 완료 비율입니다. |
| `plan_handle` | SQL 계획을 식별하는 핸들입니다. |
| `reads` | 세션에서 발생한 물리적 읽기 횟수입니다. |
| `reads(sigma)`| 모든 세션에서 발생한 총 물리적 읽기 횟수입니다. |
| `row_count` | 세션에서 처리된 행의 수입니다. |
| `sql_hash`| 현재 세션에서 실행 중인 SQL 문의 해시 값입니다. |
| `sql_param` | 세션에서 실행 중인 SQL 문의 매개변수 값입니다. |
| `sql_text`| 현재 세션에서 실행 중인 SQL 문의 전체 텍스트입니다. |
| `status` | 세션의 현재 상태입니다. 예: `running`, `suspended` 등. |
| `wait_resource` | 세션이 대기하고 있는 리소스입니다. |
| `wait_time` | 세션이 대기 중인 시간(밀리초 단위)입니다. |
| `wait_type` | 세션이 현재 기다리고 있는 대기 유형입니다. |
| `writes` | 세션에서 발생한 물리적 쓰기 횟수입니다. |
| `writes(sigma)` | 모든 세션에서 발생한 총 물리적 쓰기 횟수입니다. |

</TabItem>

<TabItem value="locktree" label="Lock tree">

| 항목 | 설명 |
| -- | -- |
| `application` | 세션을 생성한 애플리케이션의 이름입니다. |
| `client` | 세션을 생성한 클라이언트의 이름 또는 IP 주소입니다. |
| `db` | 세션이 연결된 데이터베이스의 이름입니다. |
| `elapsed_time` | 세션이 시작된 후 경과된 시간(밀리초 단위)입니다. 잠금이 발생한 이후 얼마나 시간이 흘렀는지 측정하는 데 사용됩니다. |
| `host` | 세션을 생성한 클라이언트 호스트의 이름입니다. |
| `id` | 잠금 세션의 고유 식별자(ID)입니다. |
| `last_wait_type` | 세션이 마지막으로 기다린 대기 유형입니다. |
| `row_count` | 세션에서 처리된 행(row)의 수를 나타냅니다. |
| `sql_param` | 세션에서 실행 중인 SQL 문의 매개변수 값입니다.s |
| `sql_text` | 세션에서 실행 중인 SQL 문의 전체 텍스트입니다. |
| `status` | 세션의 현재 상태를 나타냅니다. 예를 들어, `running`, `suspended`, `sleeping` 등이 있으며, 세션이 활성 상태인지, 대기 중인지를 의미합니다. |
| `user` | SQL Server에 접속한 사용자의 이름입니다. |
| `wait_resource` | 세션이 대기하고 있는 자원(resource)을 나타냅니다. |
| `wait_time` | 세션이 대기 중인 시간(밀리초 단위)입니다. |
| `wait_type` | 세션이 대기 중인 대기 유형을 나타냅니다. 예를 들어, 잠금 대기 또는 IO 대기 등의 정보를 포함합니다. |

</TabItem>

<TabItem value="process-info" label="Process information">

다음 항목은 XOS 에이전트를 설치한 경우 수집되는 지표입니다.

| 항목 | 설명 |
| -- | -- |
| `cputime` | CPU 사용 시간 |
| `cpuusage` | CPU 사용률 |
| `elapse` | CPU 사용 경과 시간 |
| `vsize` | 가상 메모리 사이즈(Kb) |
| `rss` | Resident Set Size(RSS), 프로세스와 관련한 물리적 페이지 수 |
| `state` | 프로세스 상태 |
| `ioread` | 블록을 읽는 데 소요된 실제 시간(milliseconds) |
| `iowrite` | 블록을 쓰는 데 소요된 실제 시간(milliseconds) |
| `pss` | 프로세스 고유 메모리 사용량 + 하나의 프로세스가 차지하는 공유 메모리 비율 |
| `uid` | user id |
| `cmd` | 수행 중인 명령어 |
| `longcmd` | cmd의 전체 경로 |

</TabItem>
