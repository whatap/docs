***RAC 모니터링***은 Oracle 데이터베이스에서 클러스터로 구성된 데이터베이스를 하나의 단위로 묶어 모니터링할 수 있도록 구성되었습니다. ***RAC 모니터링*** 메뉴에서 제공되는 Default 프리셋은 RAC 모니터링에 최적화되어 있습니다. 해당 프리셋을 저장, 삭제, 변경하기 위해 수정 권한이 필요합니다.

각 인스턴스에서 발생하는 주요 성능 지표들을 한눈에 비교하고 분석할 수 있습니다. 실행 중인 각 에이전트의 수, 모든 액티브 세션의 총합, 트랜잭션 수와 같은 데이터를 집계하여 효과적으로 모니터링할 수 있습니다. 사용자는 그래프 차트를 통해 시간에 따른 지푯값의 변화와 각 인스턴스의 높은 성능 지표를 쉽게 확인할 수 있습니다.

또한 가장 오랫동안 지속되는 액티브 세션에 대한 정보를 확인할 수 있어 사용자는 시스템의 성능 병목 현상을 진단하고 해결할 수 있는 근거 자료로 활용할 수 있습니다.

**멀티 인스턴스 모니터링과 차이점**

RAC 모니터링 화면은 기존의 멀티 인스턴스 모니터링에서 RAC 프리셋과 클러스터를 선택할 수 있는 인스턴스 셀렉터로 제공되었으나, 대시보드로 분리되었습니다.

* 사용할 수 있는 위젯에 차이가 있습니다.

    * [RAC] Load Balance
    * [RAC] Interconnect Bandwidth
    * [RAC] Main Waits

* Default 프리셋이 RAC 모니터링에 최적화되었습니다.
* 인스턴스 Select가 아닌 클러스터 Select로 변경되었습니다.

:::
