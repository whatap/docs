## 컬럼 정보 안내{#column-info}

| 항목 | 설명 |
| -- | -- |
| Instance Name | 인스턴스 명 |
| id | connection 식별자 |
| user | SQL 수행 계정 |
| host | 이 스레드가 연결된 hostname |
| db | default database명, 없다면 NULL |
| command | 수행 중인 명령어 type(Query/Sleep/Daemon 등) |
| time | 현재 COMMAND의 수행 시간(초) |
| state | 스레드의 현재상태 (executing/updating/sending data 등) |
| query | 실행되고 있는 statement, `Processlist.info`에서 리터럴 값을 분리한 SQL을 보여줍니다. |
| query_param | `query`에서 분리한 리터럴 값 |
| type | Thread 타입. FOREGROUND/BACKGROUND 등 |
| connection_type | Connection을 맺는데 사용하는 protocol. TCP/IP, SSL/TLS |
| query_hash | query의 해쉬값(와탭 내부 관리 값) |
| os_id | MySQL thread와 연결된 OS의 thread Id. Thread pool plugin을 사용하는 경우 NULL일 수 있습니다. |
| thread_id | thread 식별자 |
| name | 서버의 코드와 연관된 이름. 사용자 Connection 처리는 Thread/sql/one_connection 등 |
| cpu(xos) | XOS를 설치한 경우 수집되는 CPU 사용율 |
| query_id | Query ID |
| time_ms | 수행시간(milliseconds) |
| stage | 현재 프로세스가 진행 중인 단계입니다. |
| max_stage | 최대 STAGE 수(MariaDB) |
| progress | 현재 단계 내의 프로세스 진행률(0-100%). |
| memory_used | 스레드에서 사용하는 메모리(바이트)입니다. |
| max_memory_used | 스레드에서 사용한 최대 메모리(바이트)입니다. |
| examined_rows | 스레드에서 검사한 행입니다. UPDATE, DELETE 및 유사한 명령문에 의해서만 업데이트됩니다. SELECT 및 기타 문의 값은 0으로 유지됩니다. |
| holder type | holder 세션의 type |
| lock mode | holder_mode |
| waiter type | waiter 세션의 type |
| lock request | waiter_mode |
| table | holder_table/waiter_table |
| index | holder_index/waiter_index |
