<TabItem value="activesession" label="Active session">

| 항목 | 설명 |
| -- | -- |
| `user` | SQL 수행 계정 |
| `host` | 이 스레드가 연결된 hostname |
| `db` | default database명, 없다면 NULL |
| `command` | 수행 중인 명령어 type(Query\/Sleep\/Daemon 등) |
| `time` | 현재 COMMAND의 수행 시간(초) |
| `state` | 스레드의 현재상태 (executing/updating/sending data 등) |
| `query` | 실행되고 있는 statement, `Processlist.info`에서 리터럴 값을 분리한 SQL을 보여줍니다. |
| `query_param` | `query`에서 분리한 리터럴 값 |
| `type` | Thread 타입. FOREGROUND\/BACKGROUND 등 |
| `connection_type` | Connection을 맺는데 사용하는 protocol. TCP/IP, SSL\/TLS |


</TabItem>

<TabItem value="locktree" label="Lock tree">

| 항목         | 설명 |
|--------------|------|
| Instance     | 인스턴스(에이전트) 이름 |
| id           | 세션 ID |
| holder_type  | 락이 어떤 대상에 걸려 있는지를 나타냄 <br />- RECORD: 테이블의 레코드(행) 수준의 락<br />- TABLE: 전체 테이블에 대한 락 |
| holder_mode  | 홀더 세션의 락 모드 <br />- S (Shared): 공유락. 다른 트랜잭션도 읽기는 가능 <br />- X (Exclusive): 배타락. 다른 트랜잭션은 접근 불가 |
| waiter_type  | 락이 어떤 대상에 걸려 있는지를 나타냄  <br />holder_type 설명 참조 |
| waiter_mode  | 락을 대기 중인 세션의 락 모드 <br />holder_mode 설명 참조 |
| table        | 락을 대기중인 테이블 |
| index        | 락을 대기 중인 인덱스 |

</TabItem>
