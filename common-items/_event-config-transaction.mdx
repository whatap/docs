## 개요

트랜잭션 이벤트는 특정 URL, 에러, 상태 코드 등의 패턴을 조합하여 세밀하게 트랜잭션을 모니터링할 때 사용하는 기능입니다.

## 트랜잭션 이벤트 탭

트랜잭션 이벤트는 애플리케이션에서 발생하는 트랜잭션 중 특정 조건에 부합하는 경우에만 알림을 발생시킵니다. 1분 단위로 수집된 트랜잭션 데이터를 분석하여 조건에 맞는 트랜잭션 개수가 임계값을 초과하면 알림을 전송합니다.

### 이벤트 목록

이벤트 목록에서는 생성된 트랜잭션 이벤트를 확인하고 관리할 수 있습니다.

- **활성화 상태**: 스위치를 통해 이벤트 활성화 여부를 설정합니다.  
- **이벤트 레벨**: Info(회색), Warning(주황색), Critical(빨간색)으로 표시됩니다.  
- **검색 기능**: 이벤트 이름, 수신자, 대상 패턴으로 검색 가능합니다.  
- **컬럼 크기 조정**: 각 컬럼 너비를 드래그하여 조정할 수 있습니다.

#### 주요 컬럼

| 항목           | 설명                                |
|----------------|-------------------------------------|
| No             | 순번                                |
| 활성화         | 이벤트 활성화 상태                  |
| 수정           | 이벤트 편집 버튼                    |
| 이벤트 이름     | 설정한 이벤트 명칭                  |
| 조건 유형       | 경과 시간 / 에러 발생 / 패턴만       |
| 집계 기준       | 트랜잭션 카운트 집계 기준           |
| 대상 패턴       | 설정된 일치/제외 패턴               |
| 발생 횟수       | 알림 발생 기준 횟수                 |
| 일시 중지       | 알림 후 동일 이벤트 무시 시간       |
| 이벤트 수신     | 알림 수신 대상                      |

## 이벤트 추가/수정

이벤트를 추가하거나 수정할 때는 다음의 세 가지 섹션에서 조건을 설정합니다.


---

### 1. 이벤트 조건 정의

이벤트의 발생 조건을 설정하는 섹션입니다.

- **조건 유형 (필수)**
  - 경과 시간: 지정 시간(ms) 이상 소요된 트랜잭션
  - 에러 발생: 에러가 발생한 트랜잭션
  - 패턴만: 지정된 패턴에 일치하는 트랜잭션
- **경과 시간 임계값**: 조건 유형이 '경과 시간'일 경우, ms 단위로 입력  
- **집계 기준 (필수)**  
  - 에이전트별  
  - URL별  
  - 에러 클래스별  
  - 각 에이전트의 URL별  
  - 각 에이전트의 에러 클래스별  
  - 전체  
- **발생 횟수 (필수)**: 지정 횟수 이상 발생 시 알림  
- **일시 중지 (필수)**: 알림 후 동일 이벤트 재발생 시 무시 시간 (1분 ~ 1일)  
- **이벤트 동작 시간**: 특정 시간대에만 이벤트가 동작하도록 설정 가능  

---

### 2. 이벤트 대상 선택

트랜잭션 필터링을 위한 패턴을 설정합니다.

**입력 방법**

- 여러 패턴은 Enter로 구분
- 와일드카드(*) 사용 가능
- 같은 유형은 OR, 다른 유형은 AND 조건으로 적용
- 패턴이 없을 경우 전체 트랜잭션을 대상으로 함

#### 일치 패턴

- URL (예: `/api/*`, `*/users/*`)  
- 에러 클래스 (예: `*Exception`, `*Error`)  
- 에러 메시지  
- 상태 코드 (예: `4**`, `5**`)  

#### 제외 패턴

- 일치 패턴과 동일한 항목을 제외 조건으로 설정  
- 제외 패턴은 **빨간색 태그**로 표시됨  

#### 패턴 매칭 예시

```text
1. 완전 매칭
패턴: "hello"
"hello" → 일치
"Hello" → 불일치 (대소문자 구분)

2. 접두사 매칭
패턴: "hello*"
"helloworld" → 일치

3. 접미사 매칭
패턴: "*world"
"helloworld" → 일치

4. 부분 문자열 매칭
패턴: "*ell*"
"hello", "excellent" → 일치

5. 전체 매칭
패턴: "*"
"anything" → 일치

6. 복합 패턴
패턴: "hello*world", "*test*world" 등
→ 중간 문자열 매칭
