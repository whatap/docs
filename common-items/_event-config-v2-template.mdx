## 기본 이벤트 템플릿 {#basic-template}

생성한 프로젝트의 상품에 따라 기본 이벤트 템플릿을 제공합니다. 이벤트 목록의 가장 왼쪽에 <Cmdname sid="TTL06099" className="uitext" /> 버튼을 선택해 원하는 이벤트를 활성화할 수 있습니다.

제공하는 이벤트 템플릿은 다음과 같습니다. <Cmdname sid="ITM07578" className="uitext" /> 항목에 설정된 기본값은 사용자가 원하는 값으로 수정할 수 있습니다.

* ***Active Transaction***

  카테고리: [`app_counter`](metrics-app#app_counter)

  액티브 트랜잭션 발생 건수(`active_tx_count`)가 100개를 초과한 상태하면 경고(<span class="slow">Warning</span>) 수준의 알림을 보냅니다.

* ***Agent Alive Check***

  카테고리: [`agent_status_summary`](metrics-app#agent_status_summary)

  에이전트로부터 데이터를 수집하지 못할 때(`isActive` \=\= `false`) 경고(<span class="slow">Warning</span>) 수준의 알림을 보냅니다.

  :::note

  이 템플릿 이벤트는 네트워크가 끊어진 경우에도 발생할 수 있습니다.

  :::

* ***Apdex 100 by kind***

  카테고리: [`app_counter`](metrics-app#app_counter)

  에이전트 종류별 애플리케이션 성능 지표 백분률(`apdex100`)이 95% 미만이면 경고(<span class="slow">Warning</span>) 수준의 알림을, 90% 미만이면 위험(<span class="vslow">Critical</span>) 수준의 알림을 보냅니다.

* ***Apdex 100 by project***

  카테고리: [`app_counter`](metrics-app#app_counter)

  프로젝트 전체 에이전트의 애플리케이션 성능 지표 백분률(`apdex100`)이 95% 미만이면 경고(<span class="slow">Warning</span>) 수준의 알림을, 90% 미만이면 위험(<span class="vslow">Critical</span>) 수준의 알림을 보냅니다.

* ***Server CPU***

  카테고리: [`app_host_resource`](metrics-app#app_host_resource)

  호스트의 CPU 사용률(`cpu`)이 70%를 초과하면 경고(<span class="slow">Warning</span>) 수준의 알림을, 90%를 초과하면 위험(<span class="vslow">Critical</span>) 수준의 알림을 보냅니다.

* ***Server Memory***

* ***Transaction Error Count***

  카테고리: [`app_counter`](metrics-app#app_counter)

  트랜잭션 에러 건수(`tx_error`)가 10개를 초과하면 경고(<span class="slow">Warning</span>) 수준의 알림을 보냅니다.

* ***Transaction Response Time***
