* **profile_http_header_enabled** <span class='type'>Boolean</span>

  기본값 `false`

  트레이스에 HTTP 헤더 정보를 출력합니다. 헤더 수집의 여부와 무관하게 헤더를 트레이스에 노출할 것인가에 대한 설정입니다. 

* **profile_http_parameter_enabled** <span class='type'>Boolean</span>

  기본값 `false`

  트레이스 내역에 http 파라미터 정보를 기록하려면 `true`로 설정하세요. 단 파라미터는 별도 보안 키를 입력해야 조회할 수 있습니다. 보안 키는 에이전트 설치 경로의 *paramkey.txt* 파일에 6 자리로 지정합니다. *paramkey.txt* 가 존재하지 않는 경우 자동 생성되며 random 키가 설정됩니다.

* **trace_normalize_enabled** <span class='type'>Boolean</span>

  기본값 `true`

  트랜잭션 URL 을 파싱하여 정규화하는 기능을 활성화합니다.

  :::note

  값을 `false`로 변경 시 패스 파라미터 파싱이 비활성화됩니다. 이 경우 통계 데이터의 의미가 약화됨으로 디버그 용도로만 잠시 사용하는 것이 좋습니다.
  
  :::

* **trace_auto_normalize_enabled(*)** <span class='type'>Boolean</span>

  기본값 `true`

  트랜잭션 URL 정규화할 때 패턴 값을 어노테이션에서 추출하여 자동으로 파싱하는 기능을 활성화합니다.

* **trace_normalize_urls** <span class='type'>String</span>

  트랜잭션 URL을 파싱하여 정규화 합니다. 호출 URL패턴을 파싱하여 패스 파라미터를 제거합니다.  

  :::note

  예시, **/a/{v}/b**라고 선언하면 **/a/123/b** → **/a/{v}/b**로 치환합니다. 여러 개를 등록할 때는 콤마(,)를 사용합니다. **/a/*/b** 로 사용하여 특정 형식의 패스 파라미터를 제거하여 수집할 수 있습니다.
  
  :::

* **web_static_content_extensions(*)** <span class='type'>String</span>

  기본값 `js, htm, html, gif, png, jpg, css, swf, ico`

  스태틱 콘텐츠를 판단하는 확장자를 지정합니다. 여기에 설정된 확장자를 가진 트랜잭션들은 트레이스 추적과 카운팅이 제외됩니다.

* **trace_ignore_url_prefix** <span class='type'>String</span>

  트랜잭션 수집 예외 url prefix를 지정합니다. 지정한 값과 앞부분이 일치하는 url은 트랜잭션 성능을 수집하지 않습니다.
