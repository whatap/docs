
> 홈 화면 &gt; 프로젝트 선택 &gt; ![알림 아이콘](/img/ic-alert.svg) <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventSetting" className="uitext" /> <code class="classicfunc">classic</code>

애플리케이션의 **서버 자원 사용량(CPU, 메모리, 디스크)**, **진행 중 트랜잭션 수**, **트랜잭션 에러**, **응답 시간 증가** 등의 조건을 기반으로 경고 알림을 설정할 수 있습니다.

:::note

이벤트 설정 <code class="classicfunc">classic</code>은 기존 방식의 이벤트 설정을 계속 사용할 수 있도록 분리된 메뉴입니다.  
더 다양한 기능은 [이벤트 설정](/alert/event-configuration)에서 이용하실 수 있습니다.

:::

## 기본 옵션 {#default-options}

### 에이전트 비활성 경고 {#inactive-agent-alert}

설정한 시간 동안 에이전트로부터 데이터를 수신하지 못한 경우 이벤트가 발생합니다. 토글 버튼을 클릭해 기능을 켜거나 끌 수 있습니다.

- ![토글 활성화 아이콘](/img/ic-toggle-on.svg) **활성화**: 일정 기간 데이터 미수신 시 에이전트 비활성 이벤트 발생

- ![토글 비활성화 아이콘](/img/ic-toggle-off.svg) **비활성화**: 데이터 미수신 여부과 관계없이 이벤트 발생하지 않음

:::note

기존의 에이전트 비활성 확인 간격 설정은 **관리 &gt; 에이전트 목록**에서 설정할 수 있습니다. 

:::

### 이벤트 상태가 해소되면 추가 알림 {#resolved-notification}

조건이 충족되어 이벤트가 발생한 후, 상태가 정상으로 복구되면 **해소(RECOVERED) 알림**을 보낼지 여부를 선택할 수 있습니다. 토글 버튼을 클릭해 기능을 켜거나 끌 수 있습니다.

- ![토글 활성화 아이콘](/img/ic-toggle-on.svg) **활성화**: 이벤트는 상태 기반으로 동작하며, 임계치 초과 시 진행중, 임계치 이하로 복구 시 해소 상태로 전환됨

- ![토글 비활성화 아이콘](/img/ic-toggle-off.svg) **비활성화**: 임계값을 초과할 때 마다 단발성 이벤트 발생

### 에이전트 중복 알림 {#agent-duplication-alert}

동일한 이름의 에이전트가 동시에 2개 이상 기동될 경우 정상적인 모니터링이 어려울 수 있어 이벤트가 필수적으로 발생합니다. 단, 설정한 시간 동안은 중복 에이전트 이벤트가 발생하지 않습니다.

- 에이전트 이름을 중복으로 설정하지 않았는지 확인하려면 <Cmdname sid="side_eventSetting" className="uitext" /> 메뉴에서 관련 옵션을 검토하세요.

### 조건 {#condition}

항목별로 이벤트 발생 조건을 설정할 수 있으며, 항목에 따라 입력 형식이 다릅니다.

    <Tabs groupId="select-target">
    <TabItem value="server" label="서버 CPU/서버 메모리/서버 디스크">

    **조건(%)**

    <span className="slow">Warning</span>과 <span className="vslow">Critical</span> 두 단계의 임계값을 설정하세요.  
    설정값을 초과 시 경고 알림이 발생하며, <span className="vslow">Critical</span>이 우선 적용됩니다.

        - 왼쪽 값: 경고(<span className="slow">Warning</span>) 임계값
        - 오른쪽 값: 위험(<span className="vslow">Critical</span>) 임계값

    </TabItem>
    <TabItem value="transaction" label="액티브 트랜잭션/트랜잭션 에러">

        **조건(건수)**
    
        이벤트 발생 기준이 되는 건수를 입력하세요. 지정된 지속 시간 동안 설정한 건수를 초과하는 상태가 유지되면 이벤트가 발생합니다.

    </TabItem>
    <TabItem value="response" label="트랜잭션 응답시간">

    **조건(건수)**

        조건을 설정하세요. 설정한 응답시간을 초과한 트랜잭션이 지정 개수 이상 발생하면 이벤트 조건 충족합니다. 

    **조건(응답시간)**

        기준이 되는 응답시간을 선택하세요.

        - 선택 가능 시간: `1초`, `2초`, `3초`, `5초`, `10초`, `15초`, `30초`, `1분`, `2분`, `3분`, `5분`, `10분`, `30분`, `1시간`

        - 예시: 건수 기준이 **10건,** 응답시간 기준이 **1초,** 지속 시간이 **5초**인 경우 → 1초를 넘는 트랜잭션이 10건 이상 발생하고 그 상태가 5초 지속되면 이벤트가 발생합니다.

    </TabItem>
    </Tabs>

### 지속 {#continue}

선택한 시간 동안 이벤트 조건이 지속될 경우 이벤트가 발생합니다.

- 선택 가능 시간: `5초`, `10초`, `15초`, `30초`, `1분`, `2분`, `3분`, `5분`, `10분`, `30분`, `1시간`

### 무음 {#silent}

알림을 보낸 후 선택한 시간 동안 동일한 조건의 이벤트가 발생하지 않습니다. 적절한 무음시간 설정으로 과도한 이벤트 발생을 방지할 수 있습니다. 

- 선택 가능 시간: `1분`, `2분`, `3분`, `5분`, `10분`, `30분`, `1시간`, `2시간`, `3시간`, `6시간`, `12시간`, `1일`

### 이벤트 활성화 {#select-target}

토글 버튼을 클릭해 이벤트의 동작을 제어할 수 있습니다.

- ![토글 활성화 아이콘](/img/ic-toggle-on.svg) **활성화**: 설정한 조건을 충족하면 이벤트가 발생됨

- ![토글 비활성화 아이콘](/img/ic-toggle-off.svg) **비활성화**: 조건을 검사하지 않음

## 이벤트 설정 {#event-configuration}

다음 항목에 이벤트를 설정하여 시스템 상태를 실시간으로 모니터링 할 수 있습니다.

### 서버 CPU / 서버 메모리 /서버 디스크 {#server}

서버 자원 사용량이 설정한 임계값(%)을 초과할 경우 이벤트가 발생합니다. <span className="slow">Warning</span>과 <span className="vslow">Critical</span> 단계별로 다른 임계치를 설정할 수 있습니다.

- 예: CPU 사용률이 <span className="slow">Warning</span>: 80%, <span className="vslow">Critical</span>: 90%로 설정되었을 경우, 90%를 초과하면 <span className="vslow">Critical</span> 이벤트가 발생합니다.

### 액티브 트랜잭션 {#active-transaction}

현재 처리 중인 트랜잭션의 수를 모니터링합니다. 설정한 기준 건수를 초과하고, 이 상태가 **지속 시간** 이상 유지되면 이벤트가 발생합니다. 대량의 트랜잭션이 동시에 처리되고 있는 상황을 감지하는 데 유용합니다.

- 발생 건수 기본값: 100개 기준

### 트랜잭션 에러 {#transaction-error}

정상적으로 종료되지 않은 트랜잭션을 에러로 판단합니다. 에러 건수가 설정한 기준 건수를 초과하고, 해당 상태가 지속되면 이벤트가 발생합니다.

- 수집 주기: 5초

### 트랜잭션 응답시간 {#transaction-response}

각 트랜잭션이 완료되는 데 걸린 시간(응답 시간)을 기준으로 이벤트를 설정합니다. 사용자가 기대하는 응답시간을 기준으로 느려진 서비스 감지를 설정할 수 있습니다.  

- 예: 응답시간 기준을 10초, 건수 기준을 10건으로 설정한 경우  
“응답시간 10초를 넘긴 트랜잭션이 10건 이상 발생하고, 이 상태가 5초간 지속되면 이벤트 발생”
