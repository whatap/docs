## .NET 에이전트 지원 환경

에이전트를 설치하기 전에 .NET Framework 응용 프로그램에 대한 지원 환경을 확인하세요.

* **운영체제**: Windows Server 2012 이상(IIS 8.0 이상)

* **닷넷버전**: .NET Framework 4.5(4.61 이상 권장)

* **지원 라이브러리**

  * WebClient/WebRequest

  * HttpClient/HttpClientHandler

  * ASP.NET MVC 5 (IIS)

  * ASP.NET Web API 2 (IIS)

  * ASP.NET Web Forms (IIS)

  * ADO.NET (Database)

## 서버 지원 환경 분석 방법

### 분석 도구 다운로드

에이전트를 설치할 서버의 지원 환경을 잘 모른다면 다음 링크를 통해 분석 도구를 다운로드하세요.

* [https://repo.whatap.io/windows/WhaTapDotnetScan.exe](https://repo.whatap.io/windows/WhaTapDotnetScan.exe)

* [https://repo.whatap.io/windows/WhaTapDotnetScan.zip](https://repo.whatap.io/windows/WhaTapDotnetScan.zip)

### 분석 도구 사용하기

1. 다운로드한 분석 도구를 실행하세요.

    ![WhaTap System information scanner](/img/apm-dotnet-analystic-env-01.png)

1. ![숫자 1](/img/number-01.png) **분석**(**Scan**) 버튼을 클릭하세요.

1. ![숫자 2](/img/number-02.png) 영역에서 윈도우 서버의 버전이 2012 이상인지 확인하세요.

1. ![숫자 3](/img/number-03.png) 영역에서 **IIS 8.0** 버전 이상, **.NET Framework 4.5** 버전 이상인지 확인하세요.

    ![WhaTap System information scanner](/img/apm-dotnet-analystic-env-02.png)

:::note

* **.NET Framework**는 4.8.1 버전까지 지원합니다.

* **.NET Core** 버전은 개발 진행 중입니다.

:::

### 사용자의 애플리케이션 버전 확인

서버의 환경이 지원 버전을 만족하더라도 사용자의 애플리케이션이 **.NET Framework**가 아닌 **.NET Core** 버전으로 개발되었다면 .NET 모니터링을 지원하지 않습니다.

사용자의 애플리케이션 버전은 **IIS Manager**에서 다음과 같이 확인할 수 있습니다.

![IIS Manager](/img/apm-dotnet-application-pool.png)

**IIS Manager**에서 **Application Pools**를 클릭하세요. **Application Pools** 목록에서 사용하는 애플리케이션에 해당하는 풀(Pool)의 **.NET Core**가 4.0 버전인지 확인하세요.
