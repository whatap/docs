### 데이터 조회 범위에 따른 데이터

> 장기간 데이터를 조회하면 실시간 데이터가 나오지 않는 이유가 무엇인가요?

와탭은 데이터 조회 범위에 따라 표시되는 데이터 형식이 다릅니다. 조회 범위가 길어질수록 데이터 집계 간격도 길어집니다.

* **3시간 미만**: 5초 간격의 실시간 데이터

* **3시간 이상~15일 이하**: 5분 간격의 통계 데이터

* **16일 이상**: 1시간 간격의 통계 데이터


### 에이전트 데이터 수집 프로젝트 변경 {#agent-project-change}

> 에이전트 데이터 수집 프로젝트를 어떻게 변경하나요? 

와탭 에이전트 설치 디렉터리 내 *whatap.conf* 파일에서 **수집 서버 IP**와 **프로젝트 액세스 키**를 수정 후 에이전트를 재기동하세요. **수집 서버 IP** 및 **프로젝트 액세스 키**는 와탭 서비스 화면의 <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="side_agentSetting" className="uitext" /> 메뉴에서 확인할 수 있습니다. 

```bash 
whatap.server.host= # 수집 서버 IP
license= # 프로젝트 액세스 키
```

:::note

쿠버네티스 모니터링 상품의 경우 [다음 문서](/kubernetes/k8s-collection-server-ip)를 참조하세요. 

:::

#### 다중 에이전트 적용 

애플리케이션과 서버 모니터링 상품의 경우 <Cmdname sid="side_agentSetting" className="uitext" /> 메뉴에서 ***다중 에이전트 적용*** 옵션 선택 시 여러 에이전트 설정을 한 번에 변경할 수 있습니다. 

**컨테이너 환경에서 배포된 경우** 와탭 에이전트가 포함된 이미지 내의 *whatap.conf* 파일 수정 후 이미지를 재빌드 및 재배포해야 적용할 수 있습니다. UI에서 변경 시 기존 이미지 설정이 유지되므로 주의가 필요합니다.

### 이벤트 지속 시간

> 이벤트 발생 후 해소되지 않는 이유가 무엇인가요?

발생된 이벤트의 **지속(발생 횟수)** 설정을 확인해주세요. 이벤트의 지속(발생 횟수) 설정은 이벤트 발생뿐만 아니라 해소 시에도 적용됩니다. 즉, 설정된 조건을 만족하지 않으면 이벤트가 해소되지 않고 유지됩니다. 

예를 들어, 트랜잭션 에러 이벤트의 조건(건수)이 10건이고 지속 시간이 30초인 경우  
트랜잭션 에러 건수가 10건 이상으로 30초 지속되면 이벤트가 발생하고, 에러 건수가 10건 이하로 30초 지속되면 이벤트가 해소됩니다.
