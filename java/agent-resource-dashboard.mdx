## 기본 화면 

실시간 모니터링 대시보드를 통해 현재 실행 중인 에이전트의 메모리 풀 상태 및 관련 지표 변화 추이를 한눈에 확인할 수 있습니다. 위젯에서 확인할 수 있는 자바 메모리 영역 및 주요 지표는 다음과 같습니다. 

#### <Cmdname sid="TTL09121" className="uitext" />

* **Metaspace**

  클래스 메타데이터를 저장하는 공간입니다. Java 8부터 Metaspace로 변경되었으며, JVM 외부 네이티브 메모리를 사용합니다.

* **CodeHeap non-profiled nmethod / 'profiled nmethod' / 'non-nmethods'**

  JIT(Just-In-Time) 컴파일러가 생성한 머신 코드를 저장하는 공간으로 'profiled nmethod', 'non-profiled nmethod', 'non-nmethods'는 각각 컴파일된 코드를 용도에 따라 구분해 보관합니다.

* **G1 Survivor Space**
  
  G1 GC에서 살아남은 객체가 일시적으로 저장되는 영역입니다.

* **G1 Eden Space**
  
  새로 생성된 객체가 먼저 할당되는 공간입니다. G1 GC의 Eden 영역은 가비지 컬렉션 시점에 따라 Survivor 영역 또는 Old 영역으로 이동됩니다.

* **G1 Old Gen**
  
  Young GC를 거쳐 살아남은 객체가 이동되는 영역입니다. 메모리가 증가할 경우 Old Gen을 중심으로 누수가 발생하는지 확인할 수 있습니다.

* **Compressed Class Space**
  
  클래스 관련 포인터(klass pointer 등)를 압축하여 저장하는 공간입니다. Metaspace와 별도로 관리됩니다.
