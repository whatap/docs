홈 화면 > 프로젝트 선택 > ***대시보드*** > ***컨테이너 맵***

와탭 쿠버네티스의 컨테이너 맵은 쿠버네티스 모니터링의 시작점입니다. 사용자 설정 보기에 따라 각 개체는 하나의 컨테이너, 또는 하나의 Pod를 나타냅니다.

![컨테이너 맵](/img/kube-container-map-01.png)

:::info

**블록(Block)과 마이크로 블록(Micro block)**

컨테이너 맵에서는 다각형 하나를 블록(Block)이라고 합니다. 이때 에이전트가 있는 블록(블록 안에 블록)은 마이크로 블록(Micro Block)이라고 한다.

:::

## ![](/img/number-01.png) 컨테이너 또는 Pod

각 개체의 모양과 디자인은 컨테이너 또는 Pod의 특성과 상태를 나타냅니다.

* 색상 : 컨테이너 또는 Pod의 상태를 의미합니다. 상태에 대한 자세한 내용은 [다음 문서](container-status)를 참조하세요.

* 테두리

    * 개체 중에는 테두리가 표현된 것이 있습니다. 이는 컨테이너에 Java, Node.js, Python 등 마이크로 에이전트를 설치한 대상입니다. 
    * 만약 1개 Pod 내에 2개 컨테이너가 있고, 2개 중 1개 컨테이너에만 마이크로 에이전트를 설치했더라도 테두리가 표시됩니다. 즉, Pod 내 컨테이너 중에 하나라도 마이크로 에이전트가 설치되어 있다면 테두리 표시가 됩니다.

## ![](/img/number-02.png) 클러스터 프로젝트 - 네이스페이스 프로젝트 이동

![컨테이너 맵 프로젝트 목록](https://img.whatap.io/media/images/whatap_kubernetes_container_map_namespace_project_list.png)

클러스터 프로젝트 하위에 네임스페이스 프로젝트를 생성한 경우 컨테이너 맵 화면 위에 프로젝트 리스트 박스를 통해 클러스터 프로젝트를 포함한 프로젝트 목록 확인 및 프로젝트 간 이동할 수 있습니다.

:::note

* 와탭 쿠버네티스 서비스의 **클러스터 프로젝트**는 **마스터 프로젝트**의 새로운 이름입니다.
* 네이스페이스 프로젝트 생성에 관한 자세한 내요은 [다음 문서](namespace)를 참조하세요.

:::

## ![](/img/number-03.png) 사용자 설정

![사용자 설정](/img/container-map-custom-view.png)

컨테이너 맵이 제공하는 여러 사용자 설정을 통해 원하는 화면 보기를 생성할 수 있습니다. 화면 순서대로 사용할 수 있는 기능은 다음과 같습니다.

* 카테고리: ***Container*** 또는 ***Pod***를 선택할 수 있습니다.

* ***그룹화***(클러스터링): 그룹화할 수 있는 조건을 선택할 수 있습니다. 컨테이너 또는 Pod를 선택한 조건에 따라 카드로 구분해 표시합니다. 자세한 내용은 [다음 문서](container-group)를 참조하세요.

* ***ConState(state)***: 차트 지표를 선택할 수 있습니다.

* ![설정 아이콘](/img/ico-setting.svg) 버튼을 클릭하면 각 개체에 컨테이너 또는 Pod의 정보를 표시할 수 있는 옵션을 선택할 수 있습니다.

  ![옵션](/img/k8s-container-map-setting.png)

  * ***와탭 에이전트 표시하기***: 컨테이너 또는 Pod의 정보를 표시할 수 있습니다. 원하는 ***라벨*** 옵션을 선택하세요. 라벨 옵션에 대한 자세한 내용은 [다음 문서](#label)를 참조하세요.

  * ***카드 하단에 메트릭스 표시하기***: 화면 하단에 메트릭스 정보를 표시할 지 여부를 선택할 수 있습니다.

  * ***1분 내 생성된 컨테이너 표시하기***: 생성된지 1분 이내의 컨테이너의 경우 밝은 색상으로 표시하는 옵션입니다.

* ![필터 아이콘](/img/ico-filter.svg) ***필터***: 사용자가 선택한 항목에 따라 필터링해 컨테이너 또는 Pod의 상태를 확인할 수 있습니다.

* ![리셋 아이콘](/img/ico-reset.svg) ***리셋***: 사용자가 설정한 화면 보기를 초기화합니다.


### 라벨 옵션{#label}

* ***없음***: 블럭에 라벨을 표시하지 않습니다.

* ***이름***: 컨테이너/POD의 이름을 표시합니다.

* ***애플리케이션***: 컨테이너/POD 내의 애플리케이션을 표시합니다.

* ***선택된 지표***: ***ConState(state)*** 항목에서 선택한 차트 지표를 표시합니다.


## 요약 보기와 상세 보기

사용자는 관심있는 컨테이너 또는 Pod를 선택해 요약 정보 및 상세 정보를 확인할 수 있습니다. 해당 정보에는 다음과 같은 핵심 모니터링 데이터를 포함하고 있습니다.

* 트레이스 (Trace)
* 메트릭스 (Metrics)
* 로그 (Log)
* 이벤트 (Event)

이러한 데이터들을 기반으로 사용자는 쿠버네티스 환경의 Observability를 확보할 수 있습니다. 아래 **요약 보기**, **상세 보기** 화면 모두 위 4개 항목을 중심으로 정보를 제공합니다.

### 요약 보기

특정 개체 혹은 개체 집합을 클릭하세요. 오른쪽에 나타나는 **요약 보기** 화면을 통해 해당 객체의 요약 정보를 확인할 수 있습니다.

![요약 보기](https://img.whatap.io/media/images/whatap_kubernetes_container_map_right_summary.png)

### 상세 보기

**요약 보기** 화면의 오른쪽 위에 ![상세 보기 아이콘](/img/ico-detail-mode.svg) 버튼을 클릭하세요. **상세 보기** 화면으로 전환합니다.

![상세 보기](/img/container-map-detail-mode.png)

## 실시간 조회 및 과거 조회

컨테이너 맵은 기본적으로 실시간 조회 기반이지만 여러 시점에 따른 조회 기능을 제공합니다. 실시간 조회 변경은 화면 오른쪽 위에 시간 선택자를 이용하세요.

![시간 선택자](/img/kubernetes-time-seletor.png)

|조회 상태|설명|
|--|--|
|실시간 조회 (LIVE)|기본 모드<br/>10초 인터벌로 실시간 상태 변경|
|일시정지 조회 (PAUSE)|실시간 상태 변경 OFF<br/>현재 상태를 자세히 확인할 때 사용|
|과거시점 조회|사용자가 과거 특정 시점을 지정하여 확인<br/>년/월/일/시/분 지정 가능|
