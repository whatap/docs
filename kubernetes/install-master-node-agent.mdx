## 사전 환경 점검 및 설정

사용자의 쿠버네티스 환경에 따라 사전 환경 설정이 필요한 경우가 있습니다. Istio, GKE, OpenShift 환경에 해당하지 않을 경우 다음 단계로 넘어가세요.

<Tabs>

<TabItem value='istio' label='Istio 환경' default>

Istio 환경이라면 다음 명령을 실행해 필요한 자원을 생성하세요.

```bash title='SH'
kubectl create namespace whatap-monitoring
kubectl label namespace whatap-monitoring istio-injection=enabled
kubectl apply -f - <<EOF
apiVersion: networking.istio.io/v1alpha3
kind: ServiceEntry
metadata:
  name: whatap-external
spec:
  hosts:
  - whatap-proxy1
  - whatap-proxy2
  addresses:
  - 13.124.11.223
  - 13.209.172.35
  ports:
  - name: proxy
    number: 6600
    protocol: tcp
    location: MESH_EXTERNAL
EOF
```

</TabItem>

<TabItem value='gke' label='GKE 환경'>

Google Kubernetes Engine(GKE)의 경우 사용자 클러스터 생성 후 권한 설정을 위해 다음 명령을 실행하세요.

```bash
ACCOUNT=$(gcloud info --format='value(config.account)')
kubectl create clusterrolebinding owner-cluster-admin-binding \
  --clusterrole cluster-admin \
  --user $ACCOUNT
```

:::note

gcloud 명령 설치에 대한 자세한 내용은 [구글 클라우드 문서](https://cloud.google.com/sdk/docs/install)를 참조하세요.

:::

</TabItem>

<TabItem value='openshift' label='OpenShift 환경'>

![OpenShift](/img/kube-install-openshift.png)

***설치 안내*** 화면에서 ***OpenShift 환경일 경우 추가 설정*** 섹션을 열어 ***openshift_scc*** 버튼을 선택하세요. 파일을 다운로드 후 다음 명령을 실행하세요.

```bash
oc create -f whatap_openshift_scc.yaml
```

## 쿠버네티스 에이전트 생성

사용자의 쿠버네티스 버전과 Container Runtime에 맞는 *YAML* 파일을 다운로드하여 에이전트를 설치하는 단계입니다.

1. 마스터 노드에서 다음 명령어를 실행해 **VERSION**과 **CONTAINER-RUNTIME**을 확인하세요.

    ```bash
    kubectl get node -o wide
    ```

    ![](/img/k8s-check-version.svg)

1. ***에이전트 설치*** 화면에서 **VERSION**과 **CONTAINER-RUNTIME**에 맞는 *YAML* 파일을 선택하고 ***다운로드*** 버튼을 선택하세요.

    <ImgLang img='k8s-download-yaml.png' desc='yaml 다운로드' />

    :::note

    마스터 에이전트 자원 사용은 컨테이너 수 증가에 따라 영향을 받을 수 있습니다. 기본값으로 설정된 메모리 limit(350MB) 이상으로 자원을 사용할 경우 OOM(Out Of Meomory)이 발생할 수 있습니다. 사용자가 *YAML* 파일을 통해 해당 설정을 변경할 수 있습니다.

    :::

1. 다운로드한 *YAML* 파일을 마스터 노드에 업로드하세요.

1. 에이전트를 생성하기 위해 마스터 노드에서 다음 명령어를 실행하세요. 

    ```bash
    # {whatap_kube_X.YZ.yaml} 파일명을 다운로드한 파일명과 일치하도록 수정하세요.
    kubectl apply -f {whatap_kube_X.YZ.yaml}
    ```

    :::note

    **와탭 에이전트는 프로젝트 정보를 기억하기 위해 소량의 Persistent Volume(PV)가 필요합니다.** 퍼블릭 클라우드 환경의 K8S에서는 Persistent Volume Claim(PVC) 요청만으로 PV가 자동으로 생성될 수 있지만, 프라이빗 환경 기준이라면 와탭의 PVC 요청을 받을 수 있는 PV가 생성된 후에 명령어를 실행하세요.

    :::

쿠버네티스 API를 이용해 에이전트를 생성(create)한 것을 확인할 수 있습니다.

![](/img/k8s-install-agent-check.svg)


## 쿠버네티스 에이전트 설치 확인

* 쿠버네티스 에이전트 설치가 정상적으로 완료되었을 경우 다음과 같이 설치 완료 메시지를 확인할 수 있습니다.

  <ImgLang img='k8s-agent-install-check.png' desc='쿠버네티스 에이전트 설치 확인' /> 

* 다음 명령어를 실행해 마스터 에이전트와 노드 에이전트의 상태를 확인할 수 있습니다.

  ```bash
  kubectl get pod -n whatap-monitoring
  ```

  ![](/img/k8s-check-running-agent-monitoring.svg)

  **STATUS** 항목이 **Running**이라면 설치를 완료한 상태입니다. 마스터 및 노드 에이전트의 설치를 완료하면 쿠버네티스 환경의 자원 모니터링을 시작합니다. [와탭 모니터링 서비스](https://service.whatap.io)로 이동한 다음 ***프로젝트 목록***과 ***대시보드*** > ***컨테이너 맵*** 메뉴에서 성능 정보를 수집하는 것을 확인하세요. 

#### Running 상태가 아닌 경우

![](/img/k8s-not-running-agent-monitoring.svg)

마스터 노드에서 다음 명령어를 실행해 파드(Pod) 생성의 실패 원인을 파악하세요. 실행 결과 하단에 **Events:** 항목을 확인하세요.

```bash
kubectl describe pod {POD_NAME} -n whatap-monitoring
```

![](/img/k8s-check-install-fail.svg)

:::tip

`kubectl` 명령어 사용 시 `-n {네임스페이스}` 옵션을 사용하면 특정 네임스페이스를 대상으로 작업을 수행합니다. 와탭 에이전트는 기본적으로 `whatap-monitoring`이라는 네임스페이스에 설치되므로 `-n whatap-monitoring`이라는 옵션을 추가합니다.

:::

</TabItem>

## 쿠버네티스 에이전트 생성

1. 쿠버네티스 모니터링을 위한 설치 파일을 다운로드 하세요. ***whatap_kube_fargate*** 버튼을 선택하세요.

1. 다운로드한 *YAML* 파일을 이용해 에이전트를 생성하세요.

    ```bash
    kubectl -f whatap_kube_fargate.yaml apply
    ```

## 클러스터 네임스페이스 확인

에이전트 설치를 완료하고 2\~3분 후 [와탭 모니터링 서비스](https://service.whatap.io) 화면 왼쪽의 메뉴에서 프로젝트 이름을 선택하세요. 사용 중인 네임스페이스 목록을 확인 후 + 버튼을 선택해 와탭 프로젝트를 신규 생성하세요.


</TabItem>

:::note

* 애플리케이션 에이전트를 설치를 진행하려면 화면 오른쪽 아래에 ***애플리케이션 에이전트 설치*** 버튼을 선택하세요. 자세한 내용은 [다음 문서](install-application-agent)를 참조하세요.
* 에이전트를 최신 버전으로 업데이트하려면 [다음 문서](agent-update)를 참조하세요.
