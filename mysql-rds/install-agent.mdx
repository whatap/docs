## 모니터링용 계정 생성{#uid}

데이터베이스 모니터링을 하기 위해 필요한 권한을 가진 계정을 생성합니다. root(admin) 계정으로 로그인해 계정을 생성하세요.

:::note

이미 있는 계정을 사용하고 싶다면 [DB 정보 입력](#create-db-user)으로 넘어가세요. 권한이 없을 경우 모니터링을 시작할 수 없습니다.

:::

```bash showLineNumbers
create user whatap identified by 'DB_PASSWORD'; ## 원하는 password를 입력하세요.
grant process on *.* to whatap;
grant select on performance_schema.* to whatap;
grant show databases on *.* to whatap;
 
## 테이블 정보를 수집할 경우
## table list를 가져올 db명.* 또는 *.* 지정해야 table 정보를 수집합니다.
grant select on DB명.* 또는 *.* to whatap;
 
## replication 사용시
grant replication client on *.* to whatap;
grant replication slave on *.* to whatap;
 
## kill session 사용시
grant super on *.* to whatap;
```

## DB 정보 입력

![DB 정보 입력](/img/mysql-rds-db-info.png)

1. 모니터링할 대상 DB와 연결하기 위한 필수 정보를 차례로 입력하세요. <span class='vslow'>*</span> 표시는 필수 입력 항목입니다.

    * ***인스턴스 명***: DB를 식별하기 위한 명칭을 입력하세요. 프로젝트 내에서 고유한 명칭이어야 하며, 영어 소문자, 숫자, 붙임표(-)만 입력할 수 있습니다.

    * ***DB IP***: DB 서버의 IP 주소를 입력하세요.

    * ***DB Port***: 와탭 수집 서버와 연결할 포트 번호를 입력하세요.

    * ***DB User ID***: 앞서 생성한 모니터링용 계정을 입력하세요.

    * ***DB Password***: 앞서 생성한 모니터링용 계정의 비밀번호를 입력하세요.

2. 모든 항목을 입력했으면 ***연결*** 버튼을 선택하세요.

3. 와탭 수집 서버(DBCTL)에서 접속을 시도합니다. 이 과정에서 내부적으로 에이전트를 생성한 후 입력한 주소와 계정 정보로 접속을 시도하기 때문에 다소 시간이 걸릴 수 있습니다.

DB 접속을 성공하면 에이전트를 생성을 완료했다는 메시지를 확인할 수 있습니다.

* 프로젝트에 모니터링 대상 DB를 추가하려면 ***DB연결 추가*** 버튼을 선택하세요. **3. DB 정보 입력** 단계를 추가 진행합니다.

  :::note

  ***DB연결 추가*** 버튼이 보이지 않는다면 ***관리*** > ***에이전트 설치*** 메뉴로 이동한 다음 **DB 정보 입력**을 진행하세요.

  :::

* 등록한 DB 정보를 확인하려면 ***인스턴스 목록*** 또는 ***인스턴스 모니터링*** 버튼을 선택하세요.

* 에이전트 설정 내용을 확인하거나 변경하려면 ***에이전트 설정*** 버튼을 선택하세요.

* 에이전트 연결을 실패했다면 [다음 문서](troubleshooting)를 참조하세요.
