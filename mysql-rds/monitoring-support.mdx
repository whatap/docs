모니터링을 시작하기 전에 데이터베이스 지원 버전 및 에이전트 아키텍처, 설치 요건 등을 알아보세요. 

와탭 MySQL RDS 모니터링 서비스는 데이터베이스 접속에 성공하면 바로 모니터링을 시작할 수 있습니다. 해당 상품을 선택하고 프로젝트를 생성하세요. DB 접속 정보를 입력하면 DB 에이전트가 와탭 서버에 생성되어 필요한 설정을 자동 수행합니다. 설정을 완료하면 에이전트가 자동 기동해 모니터링 데이터를 수집하는 구조입니다. 그렇기 때문에 와탭 수집 서버에서 고객 DB에 접속하는 것을 허용해야 합니다. 수집 데이터 전송은 보안을 위해 SSL 통신을 사용합니다. RDS는 Public Subnet에 위치해야 하며 와탭 수집 서버의 접근을 허용해야 합니다.


## 데이터베이스 지원 버전

- **MySQL RDS** 상품은 DBX 에이전트와 DB 구간 사이에 SSL 통신을 보장하는 **AWS RDS DB**에만 사용을 권장합니다.
- **AWS RDS MySQL** 모든 버전(5.7 ~ 8.0), **AWS RDS MariaDB** 모든 버전(10.3 ~ 10.6)을 지원합니다.


## 아키텍처 구성

기존의 [MySQL 모니터링](https://docs.whatap.io/mysql/monitoring-intro) 상품은 수집 에이전트를 고객망 안에 설치해야 합니다. 반면 **MySQL RDS**는 DB에이전트(DBX)가 와탭 서버에 위치합니다. DB 접속만 가능한 환경이면 에이전트를 자동 설치하고 모니터링을 시작합니다. 이를 위해 방화벽 설정을 필수로 요구합니다.

![MySQL RDS](/img/mysql-rds-architecture.jpeg)

## 방화벽

DB 서버와 와탭 수집 서버를 연결을 위해 DB 서버에서 방화벽을 오픈하고 외부 접속 및 DB에 대한 접근을 허용해야 합니다. 따라서 모니터링 대상인 **AWS RDS DB**가 **Public subnet 위치**해야 하고 Security Group에서 **와탭 서버 접속을 허용**해야 합니다.

![방화벽](/img/mysql-rds-firewall.png)

- 소스: **54.180.11.176**
- 타겟: 모니터링 대상 DB의 Port

:::note

다음의 사항에 해당한다면 기존 [MySQL 모니터링](https://docs.whatap.io/mysql/monitoring-intro) 상품을 이용하세요.

- RDS가 private subnet에 있는 경우
- DB에 외부 접속을 허용할 수 없는 경우

:::


## 보안

와탭은 고객의 데이터를 보호하기 위해 두가지 측면의 보안을 강화했습니다. 수집 데이터 전송은 SSL 통신을 이용합니다. DB 접속 정보는 3rd-party 솔루션(Vault)을 적용해 와탭서버에 암호화하여 저장합니다.
