**DB 정보 입력** 단계를 실행하면서 연결에 실패했을 경우 다음 내용을 참고해 문제를 해결하세요.

## 연결 실패 후 자원 정리

DB 정보를 잘못 입력해 연결이 안되는 경우 다음과 같은 메시지가 나타납니다.

![DB 연결 실패](/img/mysql-rds-db-info-fail.png)

1. ***실패한 에이전트 삭제*** 버튼을 선택해 에이전트 자원을 정리하세요.

1. '***생성에 실패한 에이전트를 삭제 중입니다.***' 메시지와 함께 삭제 작업을 진행합니다.

1. 삭제를 완료한 후에는 **DB 정보 입력** 화면으로 돌아갑니다.

연결 실패 원인에 따른 메시지를 참고해 원인을 해결하세요.


## 연결 실패 사례

### IP 주소 또는 포트 오류

![IP 또는 포트 오류](/img/mysql-rds-db-fail-01.png)

* IP 주소 또는 포트 번호를 올바르게 입력했는지 확인하세요.
* 모니터링 대상 DB의 방화벽을 오픈하고 외부 접속을 허용했는지 확인하세요.

### 모니터링용 계정 입력 오류

![계정 입력 오류](/img/mysql-rds-db-fail-02.png)

* 모니터링용으로 생성한 계정의 ID와 비밀번호를 올바르게 입력했는지 확인하세요.
* 모니터링용 계정에 필수 권한을 모두 부여했는지 확인하세요.

:::note

모니터링 계정 생성에 대한 자세한 내용은 [다음 문서](install-agent#uid)를 참조하세요.

:::
