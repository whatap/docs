<Cmdname sid="side_trapSetting" className="uitext" />은 네트워크 장비에서 발생하는 SNMP Trap 알람을 실시간으로 수신·분류·관리하는 설정 메뉴입니다. SNMP Trap은 네트워크 장비에서 중요한 이벤트가 발생하면 관리 시스템으로 자동 전송되는 알림 메시지입니다.

:::note

**지원 버전**

SNMP v1/v2c와 v3 버전을 모두 지원하며, 보안 수준에 따라 Community 기반 인증과 사용자 기반 인증(USM)을 설정할 수 있습니다.

:::

## 화면 구성 및 주요 기능

<Cmdname sid="side_trapSetting" className="uitext" />은 세 개의 탭으로 구성되어 SNMP 버전별 설정을 관리합니다.

1. 관리할 NMS Manager 인스턴스를 드롭다운 메뉴에서 선택하세요. 

2. 선택된 Manager에 대한 Trap 탭을 선택하고 설정하세요.

3. [Save] 버튼을 클릭해 설정 변경사항을 저장하세요.

### Trap Reciver 탭

Trap 수신을 설정합니다. NMP Trap 메시지는 기본적으로 UDP 162번 포트를 통해 전송됩니다. 필요에 따라 다른 포트를 설정할 수 있습니다.

| 설정 항목 | 설명 |
| :--: | --- |
| **Port** (필수) | Trap 수신 포트 번호(기본값: 162) <br />  - SNMP Trap의 표준 포트 <br /> |
| **SNMP v1/v2c Trap Enable** | 토글로 SNMP v1/v2c Trap 수신 활성화/비활성화 선택 <br />  - 활성화 시 Community 기반 Trap 메시지 수신 |
| **SNMP v3 Trap Enable** | 토글로 SNMP v3 Trap 수신 활성화/비활성화 선택 <br />  - 활성화 시 USM 기반 보안 Trap 메시지 수신 |

### SNMP v1/v2c Trap Enable 탭

Community 기반 SNMP v1/v2c Trap 수신을 위한 접근 제어를 설정합니다. Trap 메시지를 수집하기 위해 필요한 커뮤니티 문자열을 설정합니다. 이 문자열은 데이터의 접근 권한을 결정합니다.

#### Community Access

| 설정 항목 | 설명 |
| :--: | -- |
| Community | Community 추가 <br /> - 새로운 Community String 입력 후 [+ Add] 버튼 클릭하여 추가 <br /> - 목록에 없는 Community를 입력해야 [+ Add] 버튼이 활성화됨 |
| Allowed Communities | 등록된 Community 목록 관리 <br /> - 허용된 Community 표시 <br /> - ![삭제 아이콘](/img/ico-trash-red.svg) 아이콘 클릭 시 해당 항목 삭제 |

### SNMP v3 Trap 탭

보안이 강화된 SNMP v3 Trap을 위한 사용자 기반 보안 모델(USM) 설정을 관리합니다.

#### SNMPv3 User Configuration(USM)

사용자를 추가 설정합니다. 다음 항목을 설정한 후, [+ Add SNMP V3 User] 버튼을 클릭해 사용자 설정 변경사항을 저장하세요.

|        설정 항목          | 설명 |
| :---------------------: | -- |
|    SNMP V3 Engine ID    | Engine ID 직접 입력 또는 검색 <br /> - [[Search Engine ID](#search-engine-id)] 버튼 클릭 시 검색 팝업 표시  |
|        Username         | SNMP v3 사용자명 입력 |
|     Security Level      | 보안 단계 선택 <br /> - **noAuthNoPriv**: 인증 없음, 암호화 없음 <br /> - **authNoPriv**: 인증만 사용 <br /> - **authPriv**: 인증과 암호화 모두 사용 |
| Authentication Protocol | 인증 프로토콜 선택(MD5, SHA1, SHA256 등) <br /> - Security Level이 authNoPriv 또는 authPriv일 때 활성화 <br /> - 인증 키 입력 필드 제공 |
|    Privacy Protocol     | 암호화 프로토콜 선택(DES, AES, AES128, AES256 등) <br /> - Security Level이 authPriv일 때 활성화 <br /> - 암호화 키 입력 필드 제공 |

##### Engine ID 검색 {#search-engine-id}

SNMP v3 장비의 Engine ID를 검색할 수 있습니다.

1. **SNMP V3 Engine ID** 입력창 옆의 [Search Engine ID] 버튼을 클릭하세요. 

1. Search Engine ID 팝업창에서 검색할 엔진의 항목을 입력하세요.

    |     항목      | 설명                                                                            |
    | :-----------: | ------------------------------------------------------------------------------- |
    |   **Host**    | 대상 장비의 IP 주소 입력                                                        |
    |   **Port**    | SNMP 포트 번호(기본값: 161)                                                     |
    | **Community** | SNMP Community String(기본값: public) <br /> - v1/v2c를 사용하여 Engine ID 검색 |

1. [Search] 버튼을 클릭해, 입력한 정보로 Engine ID 검색을 실행합니다. 

    * 검색 성공 시 자동으로 Engine ID 필드에 입력합니다.

1. 검색 취소 및 팝업을 닫고 싶다면 [Cancel] 버튼을 클릭하세요.
