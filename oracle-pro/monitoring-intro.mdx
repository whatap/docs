## Oracle Pro 모니터링 특장점

와탭의 Oracle Pro 모니터링 상품은 기존 쿼리(Query) 기반의 데이터 수집 방식을 한 단계 뛰어넘는 혁신을 제시합니다. 기존의 쿼리 기반 방식은 데이터베이스 서버에 상당한 부하를 주며, 이는 시스템 성능 저하로 이어질 수 있습니다. 반면, Direct Memory Access(DMA) 기술을 활용한 와탭의 접근 방식은 서버의 메모리에 직접 접근하여 필요한 데이터를 추출함으로써, 데이터베이스 서버에 부하를 거의 또는 전혀 발생시키지 않습니다. 이는 특히 고성능을 요구하는 데이터베이스 환경에서 큰 장점으로 작용합니다.

또한, 쿼리 기반 수집 방식은 데이터 갱신 주기가 느리고, 특정 쿼리 실행 시점에만 데이터를 수집할 수 있습니다. 이에 비해 Oracle Pro 모니터링은 메모리 접근을 통해 데이터를 거의 실시간으로 수집할 수 있으며, 최소 0.01초 간격으로 세션 및 SQL 성능 데이터를 수집할 수 있습니다. 이는 데이터베이스 관리자가 시스템의 성능을 더욱 세밀하게 모니터링하고, 문제가 발생하는 즉시 대응할 수 있게 합니다.

Oracle Pro 모니터링 상품의 또 다른 중요한 특징은 시스템 장애나 Hang 상황에서도 데이터 수집이 중단되지 않습니다. 전통적인 쿼리 기반 방식에서는 이러한 시나리오에서 데이터 수집이 어려울 수 있습니다. Oracle Pro를 사용함으로써, 데이터베이스 관리자는 어떠한 상황에서도 데이터의 연속성을 보장받으며, 이를 통해 시스템의 안정성과 가용성을 극대화할 수 있습니다.

결국, 와탭의 Oracle Pro 모니터링 상품은 데이터베이스 모니터링의 효율성과 정확성을 향상시키는 동시에, 시스템 부하를 최소화하고, 비즈니스 연속성을 보장하는 데 필수적인 기술입니다. 이는 기업이 더 빠르고, 더 정확하며, 더 신뢰성 높은 의사결정을 할 수 있는 기반을 마련해 줍니다.

* **메모리 접근 방식**

  와탭의 Oracle Pro는 **Direct Memory Access**(DMA) 기술을 통해 DB 서버 메모리에 직접 접근해 데이터를 수집합니다. 기존 쿼리 방식의 한계를 뛰어넘어 서버 부하 없이 빠르고 효율적인 데이터 수집을 제공합니다.

* **부하없는 모니터링 정보 수집**

  Oracle Pro는 DB 서버 메모리에 직접 접근해 데이터를 추출하며, 1초에 20번 성능 정보를 수집해도 서버 부하를 거의 발생시키지 않습니다. 이는 고성능 DB 환경에서도 안정적이고 효율적인 모니터링을 지원합니다.

* **정밀한 SQL 성능 데이터 수집**

  Oracle Pro는 1초에 20번 데이터를 수집해 정밀한 분석이 가능하며, 기존 쿼리 기반 방식의 5초 간격 SQL 수집보다 훨씬 세밀한 정보를 제공합니다. 이를 통해 관리자는 실시간으로 문제를 파악하고 즉각적으로 대응할 수 있습니다.

* **DB Hang 상태에서도 모니터링 분석 가능**

  DB 장애나 Hang 상황에서도 Oracle Pro는 데이터 수집이 중단되지 않습니다. 이는 데이터 연속성을 보장하고, 장애 상황에서도 분석 및 대응이 가능하게 하여 시스템 안정성과 가용성을 극대화합니다.
