## 세션 정보 확인하기 {#checksessioninfo}

### <Cmdname sid="session_detail" /> {#session-details}

다음 메뉴 경로에서 세션 항목을 선택하면 확인할 수 있는 <Cmdname sid="session_detail" className="uitext" /> 창에 대해 안내합니다.

* <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_dbMonitoring" className="uitext" />

* <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_multiInstance" className="uitext" />

* <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_dbTrendComparison" className="uitext" />

* <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_dbLockTree" className="uitext" />

* <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_dbWaitAnalysis" className="uitext" />

세션 테이블 목록에서 세션 항목을 선택하면 <Cmdname sid="session_detail" className="uitext" /> 창이 나타납니다. 선택한 세션에 대한 자세한 내용을 확인할 수 있습니다.

<InDoc product="oracle-pro,oracle"><ImgLang img='db-session-details-info-oracle-pro.png' desc='세션 상세 정보' /></InDoc>

* 세션에 대한 정보는 기본값으로 실시간 조회되고 있습니다. 시간 선택자에서 ![정지 아이콘](/img/ico-pause.svg) 버튼을 선택하면 실시간 조회를 중지하고, 중지한 시간 기준의 데이터를 확인할 수 있습니다. 실시간 조회를 중지한 상태에서 데이터를 갱신하려면 ![새로고침 아이콘](/img/ico-refresh.svg) 버튼을 선택하세요.

* ***Kill Session***: 선택한 세션을 중지할 수 있습니다. <Cmdname sid="session_kill" className="uitext" /> 창이 나타나면 비밀번호(***Parameter Key***) 값을 입력한 다음 <Cmdname sid="apply" className="uitext" /> 버튼을 선택하세요.

  :::note

  실시간 조회 모드에서만 이 기능을 이용할 수 있습니다.

  :::

* <Cmdname sid="TTL06721" className="uitext" />: 선택한 세션 기록을 확인할 수 있는 <Cmdname sid="side_sessionHistory" className="uitext" /> 메뉴로 이동할 수 있습니다.

  :::note

  <Cmdname sid="side_sessionHistory" className="uitext" /> 메뉴에 대한 자세한 내용은 <Link to="analysis-session-history">다음 문서</Link>를 참조하세요.

  :::

* ***Session Stat***: 선택한 세션의 다양한 성능 지표 및 통계 정보를 제공합니다. 세션에서 수행된 작업의 유형과 횟수, 리소스 사용량 등을 포함합니다.

* ***Delta*** / ***Sigma***: 선택한 세션의 변화율과 통계적 분석 정보를 제공합니다. 세션의 활동에 대한 변화량을 확인할 수 있습니다.

* ***Session Info***: 선택한 세션에 대한 기본 정보와 상태가 포함된 상세 내용을 확인할 수 있습니다.

* ***Wait Info***: 선택한 세션의 대기 이벤트 및 대기 시간 정보를 제공합니다. 특정 작업을 실행하기 위해 데이터베이스 시스템에서 대기할 때 발생하는 대기 이벤트에 대한 정보도 확인할 수 있습니다.

* ***SQL Text*** / ***Plan***: 선택한 세션의 SQL 쿼리문과 Plan 정보를 확인할 수 있습니다. 자세한 내용은 [다음 문서](#query-details)를 참조하세요.

### <Cmdname sid="TTL06224" /> {#query-details}

현재 수행 중인 세션 또는 과거 세션의 SQL 쿼리 정보를 확인하려면 <UI>query_text</UI> 컬럼 항목을 확인하세요. SQL 쿼리문과 Plan 정보를 확인할 수 있는 <Cmdname sid="TTL06224" className="uitext" /> 창이 나타납니다.

<ImgLang img="db-sql-details-new-oracle-pro.png" desc="SQL 상세" />

* <Cmdname sid="BTN06225" className="uitext" />: 해당 쿼리를 기준으로 통계 데이터를 조회할 수 있는 <Cmdname sid="side_dbSqlStat" className="uitext" /> 메뉴로 이동할 수 있습니다.

* ***Formatting***: SQL 쿼리문에 들여쓰기 및 포맷팅을 적용해 가독성을 높일 수 있습니다.

* ***Runtime Plan***: 선택된 SQL 쿼리의 실행 계획과 런타임 정보를 제공합니다. 실행 횟수, 평균 실행 시간, 평균 물리적 읽기 등 세부 정보를 제공합니다.

  * <Cmdname sid="select_1_hour" className="uitext" /> / <Cmdname sid="select_1_day" className="uitext" />: 최근 1시간 또는 1일 기준으로 통계 정보를 제공합니다.

* ***Explain Plan***: 옵티마이저가 예측한 실행 계획을 보여줍니다. 비용, 작업, 객체 이름, 카디널리티 등의 정보를 제공합니다.

* ***Plan History***: 데이터베이스에서 실행된 SQL 쿼리의 실행 계획에 대한 이력을 확인할 수 있습니다.

* ***Bind Capture***: 데이터베이스에서 실행된 SQL 쿼리에 사용된 바인드 변수의 값을 확인할 수 있습니다. 이를 통해 쿼리 실행의 실제 내용을 확인할 수 있습니다.

  :::note

  실시간 실행된 bind 값이 아닌 데이터베이스에 캡처된 값(`v$sql_bind_capture`)입니다. 건수가 많으면 100개까지 표시합니다.

  :::

### 튜닝 가이드 {#tunning-guide}

<Cmdname sid="TTL06224" className="uitext" /> 창의 Plan 트리에서 튜닝 가이드를 확인할 수 있습니다. ![튜닝 아이콘](/img/ic-chat-red.svg) 버튼을 선택하세요. 제공하는 튜닝 가이드를 참고하여 데이터베이스 최적화를 진행하세요.

<ImgLang img="db-sql-details-tunning-guide-oracle-pro.png" desc="Tunning guide" />

:::note

튜닝 가이드는 Plan Tree의 노드에 `FULL` 또는 `CARTESIAN` 키워드를 포함할 경우 확인할 수 있습니다.

:::
