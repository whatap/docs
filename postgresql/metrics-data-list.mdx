데이터베이스 메트릭은 데이터베이스의 작업량과 리소스 사용량을 포괄적으로 수집합니다. 와탭 모니터링 서비스는 지속적으로 메트릭을 수집하여 데이터베이스의 병목 현상을 확인하고, 성능 문제의 근본 원인을 식별하는 데 도움이 되는 데이터를 제공합니다.

그외 지표에 대한 자세한 내용은 [다음 링크](https://www.postgresql.org/docs/)를 참조하세요.

<div class='db-metric'>

| Metric | Description |
| -- | -- |
| `active_sessions` | 쿼리를 실행 중인 세션 수 |
| `blk_read_time` | 해당 데이터베이스의 백엔드에서 데이터 파일 블록을 읽는 데 소요한 시간(milliseconds), `track_io_timing`이 활성화한 경우, 그렇지 않으면 `0` |
| `blk_write_tiem` | 해당 데이터베이스의 백엔드에서 데이터 파일 블록을 작성하는데 소요한 시간(milliseconds), `track_io_timing`이 활성화한 경우, 그렇지 않으면 `0` |
| `blks_hit` | 디스크 블록이 버퍼 캐시에서 이미 발견되어 읽기가 필요하지 않은 횟수(운영체제의 파일 시스템 캐시가 아닌 PostgreSQL 버퍼 캐시의 히트만 포함됨) |
| `blks_hit_ratio` | 버퍼 캐시 히트율<br/>sum(blks_hit)*100/sum(`blks_hit` + `blks_read`) |
| `blks_read` | 해당 데이터베이스에서 읽은 디스크 블록의 수 |
| `buffers_checkpoint` | 체크포인트(`check_point`) 동안 기록된 버퍼 수 |
| `check_point` | `check_point`는 모든 데이터 파일이 순차적인 트랜잭션 로그 가운데, 한 시점을 기준으로 모든 데이터 파일이 물리적으로 모두 쓰여진 한 시점을 의미. `check_point`가 일어난 수 |
| `checkpoints_write_time` | 파일이 디스크에 기록되는 체크포인트 처리 부분에서 소요된 총 시간(milliseconds) |
| `checkpoints_tiemed` | 예약되어 수행된 체크포인트 수 |
| `commit_count` | 커밋된 트랜잭션의 수 |
| `conflicts` | 해당 데이터베이스에서 복구 중 충돌로 인해 취소된 쿼리 수 |
| `connect_error` | 서버 연결 실패 횟수 |
| `deadlock_count` | 해당 데이터베이스에서 감지된 교착 상태(deadlock) 수 |
| `dml_count` | DML(Data Manipulation Language) 문법(statement)이 수행된 횟수 |
| `fatal_count` | 치명적인 오류로 종료된 데이터베이스 세션 수 |
| `index_hit_ratio` | 캐시 내 인덱스 비율<br/>(sum(`idx_blks_hit`) - sum(`idx_blks_read`)) / sum(`idx_blks_hit`) * 100 |
| `lock_wait_sessions` | 현재 row lock을 획득하기 위해 대기중인 수 |
| `long_running_sessions` | 현재 데이터베이스에서 무기한으로 실행되는 쿼리의 수 |
| `max_connections` | 최대 동시 클라이언트 연결 수, 커넥션 수가 부족할 경우 **Too ManyConnection Error**가 발생할 수 있습니다. |
| `numbackends` | 해당 데이터베이스에 연결된 백엔드 수 또는 공유 개체의 경우 `nULL` |
| `rollback_count` | 해당 데이터베이스에서 롤백된 트랜잭션의 수 |
| `table_hit_ratio` | 테이블의 blks hit(캐시 히트)와 blks read(디스크 읽은 횟수)의 비율 |
| `temp_bytes` | 해당 데이터베이스의 쿼리에 의해 임시 파일에 기록된 총 데이터 양 |
| `temp_files` | 해당 데이터베이스의 쿼리에 의해 생성된 임시 파일 수 |
| `total_runtime` | 퀴리를 실행 중인 세션들의 실행 시간 합 |
| `total_sessions` | 모든 세션 수 |
| `tup_deleted` | 해당 데이터베이스에서 쿼리에 의해 삭제된 행 수 |
| `tup_fetched` | 해당 데이터베이스에서 쿼리가 가져온 행 수 |
| `tup_inserted` | 해당 데이터베이스에서 쿼리에 의해 삽입된 행 수 |
| `tup_returned` | 해당 데이터베이스에서 쿼리에서 반환된 행 수 |
| `tup_updated` | 해당 데이터베이스에서 쿼리에 의해 업데이트된 행 수 |
| `uptime` | 서버가 시작된 후 지나간 시간 |
| `vacuum_running_count` | 수행 중인 vacuum 기능의 수 |
| `wait_sessions` | 대기 중인 세션 수 |
| `warning_count` | 총 오류 및 경고, 참고 수 |
| `cpu(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `cpu_idle(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `cpu_iowait(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `cpu_sys(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `cpu_user(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `mem(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `mem_available(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `mem_buffcache(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `mem_free(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `mem_swapfree(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `mem_swaptotal(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `mem_total(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `mem_used(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `net_recv_bytes(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `net_recv_packets(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `net_send_bytes(xos)` | XOS 에이전트에서 수집하는 OS 지표 |
| `net_send_packets(xos)` | XOS 에이전트에서 수집하는 OS 지표|
