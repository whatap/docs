에이전트를 설치한 다음 확인해야할 사항입니다. 만약 설치에 문제가 생겼다면 [다음 문서](agent-troubleshooting)를 참조하세요.

## 에이전트 로그

애플리케이션 서버가 실행되면 애플리케이션의 모니터링 정보를 수집하기 시작합니다.

로그의 위치는 `$WHATAP_HOME`의 *logs* 디렉터리에 다음과 같은 파일로 생성됩니다. 로그 파일의 종류는 두 가지입니다.

* *whatap-boot-```yyyymmdd```.log* : 데이터 수집 로그
* *whatap-hook.log* : 데이터 수집 로그

정상 동작하는 로그의 예시는 다음과 같습니다. 

```javascript title=SH
_      ____       ______Python-AGENT
| | /| / / /  ___ /_  __/__ ____
| |/ |/ / _ \/ _ `// / / _ `/ _ \
|__/|__/_//_/\_,_//_/  \_,_/ .__/
                          /_/
Just Tap, Always Monitoring
WhaTap Python Agent version 0.1.33, 20171017

WHATAP_HOME: /Users/kimjihye/workspace/whatap/python-sample/whatap
Config: /Users/kimjihye/workspace/whatap/python-sample/whatap/whatap.conf
Logs: /Users/kimjihye/workspace/whatap/python-sample/whatap/logs
```

## GraphQL 성능 수집

GraphQL 라이브러리가 웹 트랜잭션에 포함될 경우 Document, Latency를 수집합니다.

도큐먼트별 성능 분석이 필요하면 graphql api endpoint를 트랜잭션 예외로 설정하세요. 설정 후 자동으로 `graphql.execute` 함수를 트랜잭션 시작점으로 합니다. 도큐먼트 최상단 객체의 이름을 서비스 이름으로 트랜잭션을 수집합니다.
