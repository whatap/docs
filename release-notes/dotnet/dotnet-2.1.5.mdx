2023년 9월 13일

- <Status>Fixed</Status> Windows Communication Foundation(이하 WCF)과 웹 서비스의 트랜잭션 종료 시점을 간헐적으로 찾지 못하는 버그 수정

- <Status>New</Status> <code>webservice_method_timeout</code> 옵션 추가

  WCF와 웹 서비스 내에서 Exception 핸들링이 되지 않은 경우 트랜잭션 종료 시점을 알 수가 없습니다. 이런 경우를 대비해 WCF와 웹 서비스는 기본적으로 30초가 지나면 타임아웃 처리됩니다.
  
  <code>webservice_method_timeout</code> 옵션 값을 변경하면 타임아웃 시간을 조절할 수 있습니다. 단위는 밀리세컨드(millisecond, ms)입니다. 자세한 내용은 <Link to='../../dotnet/agent-webservice'>다음 문서</Link>를 참조하세요.
