2023년 10월 16일

- <Status>Changed</Status> 하나의 트랜잭션에 너무 많은 스텝이 발생하여 부하를 발생시키지 않도록 1000개 이상 넘어가는 스텝은 무시하도록 수정

- <Status>Fixed</Status> 비동기식 호출 방식에서 HTTP request body가 표시되지 않는 현상 수정


- <Status>Fixed</Status> 일정 시점 이후 HTTP header가 비정상적으로 표시되는 버그 수정

- <Status>Fixed</Status> 같은 네임 스페이스에 있는 Windows Communication Foundation(WCF, 웹서비스)와 상관없는 퍼블릭 메소드가 트랜잭션을 유발하여, 트랜잭션 추적이 비정상 종료되는 오류 수정

- <Status>Fixed</Status> 에이전트 삭제 시 IIS 메모리에 로딩된 일부 파일이 삭제 후에도 남아 있는 오류 수정

- <Status>Deprecate</Status> WCF(웹서비스) 타임아웃 시간 설정 옵션(<code>webservice_method_timeout</code>) 제거

  - WCF 종료 시점을 캡쳐하지 못하여 타임아웃이 발생하는 오류 수정

  - 다만 사용자가 WCF 메소드 안에서 Exception 처리를 제대로 하지 않을 경우, 트랜잭션 로스트가 발생할 수 있습니다. <Link to='../../dotnet/agent-transaction'>다음 문서</Link>에서 `trace_active_transaction_lost_time` 옵션을 참조하세요.
