2024년 10월 02일

- <Status priority='0'>Fixed</Status> 최근 추가된 모니터링 대상 중 CancellationToken 관련 메소드 임시 삭제

- <Status priority='1'>New</Status> .NET Core에서 `expand_transaction_level` 설정 지원 추가

  - **expand_transaction_level** <span class='type'>Boolean</span>
    
    기본값 `false`
    
    해당 옵션을 `true`로 설정하면 트랜잭션 외부에서 발생하는 SQL 및 HTTP 호출을 트랜잭션으로 간주하여 히트맵에 표시합니다.
    
  - **expand_transaction_basetime** <span class='type'>Int</span>
    
    기본값 `1000`ms (1초)
    
    `expand_transaction_level` 옵션이 `true`로 설정된 경우, 지정한 시간보다 응답 시간이 오래 걸릴 요청만 데이터를 수집합니다. 초당 최대 100건의 데이터를 수집합니다. 최소값은 `100` ms로 0.1초입니다.

- <Status priority='1'>New</Status> Desktop application 모니터링 지원 추가

  - `expand_transaction_level` 옵션의 값을 `true`로 설정하세요.

  - 어플리케이션 실행 전에 다음 명령을 프롬프트에 입력하세요.
    
    ```csharp
    SET COR_PROFILER={D76F1D76-A9E0-4C87-874F-C0AD93D4229B}
    SET COR_ENABLE_PROFILING=1

    SET CORECLR_PROFILER={21CAE18A-4E44-4578-83FD-0576AAA47E68}
    SET CORECLR_ENABLE_PROFILING=1
    SET DOTNET_STARTUP_HOOKS=C:\Program Files\WhaTap .NET\core\Whatap.Startup.NetCore.dll

    SET WHATAP_TRACE_IIS_ONLY=0
    SET EXPAND_TRANSACTION_LEVEL=1
    SET EXPAND_TRANSACTION_BASETIME=100
    ```
    
  - 명령어 실행 후 *Desktop-Monitoring.exe* 대신 대상 애플리케이션을 실행하세요. 

  :::note 

   Desktop application 모니터링 지원은 베타 단계로 모니터링 범위 및 사용 편의성을 점진적으로 업그레이드할 예정입니다.

  :::
