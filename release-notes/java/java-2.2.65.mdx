- <Status priority='1'>Feature</Status> SQL 바인드 변수 길이 초과 시 표시 방식 옵션 추가

    - `trace_sql_param_show_length_enabled`

    ```

    ```

- [Feature] SQL 바인드 변수 길이 초과 시 표시 방식 옵션 추가
  - `trace_sql_param_show_length_enabled=true`
    - 기본값: `false`
  - `trace_sql_param_length`
    - 기본값: `40`
  - 동작: 바인드 변수가 기본값(40자) 초과하면 `…(총 글자수)`로 표기

- [Feature] lettuce 5.x: 연결 생성 실패 시 에러 추적

- [Feature] lettuce 5.x: 비동기 명령 에러 측정 지원

- [Feature] Spring Boot 2.4.x 이하: WebClient 호출 취소(cancel) 추적



- <Status priority='0'>Fixed</Status> 동일한 Linux host에서 여러 개의 docker를 수행하는 경우 vCPU 중복 집계 방지하도록 수정

- <Status priority='0'>Fixed</Status> spring-boot-3.2 이상 버전에서 WebClient로 opentelemetry 에이전트를 사용하는 애플리케이션 연계 추적 시 멀티 트랜잭션탭에서 연결되지 않는 현상 수정

- <Status priority='0'>Fixed</Status> Java 1.7 미만인 경우 jmx를 통해 system cpu 지표를 수집하는 현상 방지

    - 에이전트 로그에 sigar 라이브러리 사용하도록 표시

    ```
    # default: false
    sigar_enabled=true
    ```
