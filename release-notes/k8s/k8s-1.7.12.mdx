2024년 09월 30일

* <Status priority='1'>New</Status> 태그 카운트 `kube_process` 카테고리 추가

  노드(Node)에서 실행 중인 쿠버네티스 관련 프로세스를 모니터링할 수 있는 태그 카운트를 추가했습니다. 이를 통해 특정 노드에서 동작 중인 프로세스를 추적하고, 성능 지표를 수집할 수 있습니다.
  
  :::note

  * 관련 에이전트 옵션(`collect_kube_node_process_metric_target_list`)에 대한 자세한 내용은 [다음 문서](../../kubernetes/set-agent/#collect_kube_node_process_metric_target_list)를 참조하세요.
  
  * 프로세스(`kube_process`) 카테고리에 대한 자세한 내용은 [다음 문서](../../kubernetes/metrics-kubernetes/#kube_process)를 참조하세요.

  :::

- <Status priority='1'>New</Status> **whatap-control-plane-helper** 컨테이너에 `kube_client_tls_verify` 옵션 추가

  :::note

  옵션값을 `false`로 설정하면 TLS 인증을 강제하지 않고, 인증 절차를 생략할 수 있습니다. 자세한 내용은 [다음 문서](../../kubernetes/set-agent/#kube_client_tls_verify)를 참조하세요.

  :::

- <Status priority='1'>Feature</Status> 태그 카운트 `kube_stat` 카테고리에 노드(Node) 상태별 카운트 지표 추가

  클러스터 내 노드 상태에 따른 카운트 지표를 추가했습니다. 이는 노드의 Conditions(type:Ready) 필드를 기반으로 집계된 노드의 수를 보여줍니다.

  - `ready_node`: **Ready**의 status가 `True`로 설정된 노드의 수입니다.

  - `not_ready_node`: **Ready**의 status가 `False`로 설정된 노드의 수입니다.

  - `unknown_node`: **Ready**의 status가 `Unknown`으로 설정된 노드의 수입니다.

  :::note
  
  정수 단위로 표현되며, 노드 상태에 따라 각 상태별 노드 수를 집계하여 클러스터 내 노드 상태를 모니터링합니다.

  :::

- <Status priority='1'>Feature</Status> 태그 카운트 `kube_pod_stat` 카테고리에 Pod 상태(Phase)별 카운트 지표 추가

  클러스터 내 Pod 상태(Phase)에 따른 카운트 지표를 추가했습니다. 이를 통해 현재 클러스터에서 실행 중인 Pod의 상태(Phase)를 빠르게 파악할 수 있습니다.

  - `running_pod`: 현재 클러스터에서 실행 중인 Pod의 수입니다.

  - `succeeded_pod`: 정상 완료된 Pod의 수입니다.

  - `failed_pod`: 실패한 Pod의 수입니다.

  - `pending_pod`: 아직 실행되지 않은 Pod의 수입니다.

  - `unknown_pod`: 상태가 불명확한 Pod의 수입니다.

  :::note

  정수 단위로 표현되며, Pod 상태(Phase)에 따라 각 상태별 Pod의 수를 집계하여 클러스터 내 Pod 상태를 모니터링합니다.

  :::
