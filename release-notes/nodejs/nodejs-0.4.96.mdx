2024년 05월 30일

- <Status>Feature</Status> 서버에서 클라이언트로 데이터 전송 시 데이터 수집을 위한 websocket 패키지 모니터링 추가

- <Status>Feature</Status> 에이전트 설정 옵션 추가

  - **ignore_nextjs_build_file_enabled** <span class='type'>Boolean</span>

    기본값 `true`

    *Next.js* 빌드 파일에 대한 모니터링 여부를 설정합니다. 이 옵션값을 `true`로 설정하면 *Next.js* 빌드 파일을 모니터링하지 않습니다.

  - **ignore_nextjs_build_file_path** <span class='type'>String</span>

    기본값 `/next/`
    
    모니터링에서 제외할 *Next.js* 빌드 파일의 시작 경로를 지정할 수 있습니다. `ignore_nextjs_build_file_enabled` 값이 `true`일 경우에 적용됩니다. 쉼표(,)를 구분자로 여러 개의 경로를 설정할 수 있습니다.

    ```ini title="whatap.conf"
    ignore_nextjs_build_file_path=/next/abc/,/next/def/
    ```

- <Status>Changed</Status> `trace_sampling_enabled` 옵션의 기본값을 `true`로 변경

- <Status>Fixed</Status> HTTP 모니터링에서 socket 오류에 대한 통계 데이터 수집을 무시하도록 수정

- <Status>Fixed</Status> 트랜잭션의 결과가 <b>Lost Connection</b>일 경우 URL이 표시되지 않는 문제 수정

