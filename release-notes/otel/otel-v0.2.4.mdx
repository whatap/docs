## Features

### 1. HTTP Status 오류 표시
- **설정 키**: `transaction_status_error_enable`  
- **설명**: HTTP Status 코드가 400 이상일 경우 오류로 표기  
- **기본값**: `false`

### 2. HTTP Status Code 통계 수집
- **전체 URL 단위 Status Code별 건수 수집**  
  - 설정 키: `tx_status_meter_enabled`  
  - 주기: 10초  
  - 기본값: `true`
- **URL별 Status Code 건수 수집**  
  - 설정 키: `stat_txstatus_enabled`  
  - 주기: 5분  
  - 기본값: `true`

### 3. 분산 추적용 HTTP call span 정보 강제 설정
- **설명**: HTTP call span 정보가 없을 때 가상 span을 생성하여 분산 추적 연계  
- **설정 키**  
  - `otel_virtual_httpc_enabled` (기본값: `false`)  
  - `otel_virtual_httpc_name` (기본값: `"external http call"`)

### 4. Gateway 관련 정보 수집
- **게이트웨이 정보 수집 활성화**  
  - 설정 키: `otel_gateway_enabled`  
  - 기본값: `false`
- **게이트웨이 경과 시간 속성 키**  
  - 설정 키: `otel_attr_tx_gw_time_keys`  
  - 기본값: `"duration.kong"`
- **API 호출 경과 시간 속성 키**  
  - 설정 키: `otel_attr_tx_api_time_keys`  
  - 기본값: `"duration.proxy"`
- **수집 항목 및 주기**  
  - 전체 URL 단위 트랜잭션·게이트웨이·API 호출 경과 시간 수집 (10초)  
  - URL·Status별 트랜잭션·게이트웨이·API 호출 경과 시간 수집 (5분)
- **응답시간 대체 기능**  
  - 설정 키: `otel_replace_tx_resp_time_enabled` (기본값: `false`)  
  - 대체에 사용할 속성 키: `otel_attr_replace_tx_resp_time_keys` (기본값: `"duration.kong"`)

## Fixed

- **HTTP Method 무시 설정 오류 수정**  
  - 기본적으로 `PATCH`, `OPTIONS`, `HEAD`, `TRACE` 메소드는 트랜잭션으로 수집되지 않음
