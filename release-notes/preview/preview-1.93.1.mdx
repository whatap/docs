### 컨테이너맵

- <Status>New</Status> 쿠버네티스 상품군의 실험실 메뉴에 <UI>Pending Pod</UI> 기능 추가
- <Status>Change</Status> 한 개의 Pod 내에 1개 이상의 컨테이너가 존재할 경우 컨테이너 정보를 노출하도록 수정
- <Status>Feature</Status> Job형태의 Pod에 추가 로그 정보를 제공하기 위해 연관이벤트에 CronJob 추가

### Flex 보드

  * 로그 4xx, 5xx 건수
  * 로그 요청 성공률
  * 로그 응답시간

### 정비 계획

<Status>Change</Status> 화면 상단 정비 계획 안내 문구 스타일 변경 및 버튼 위치 수정

### 로그

<Status>Change</Status> 로그 설정 메뉴에서 데이터 유지기간 설정 기능을 '카테고리별 로그 데이터 유지기간' 설정으로 수정

### 멀티 트랜잭션 추적

<Status>Fixed</Status> <UI>분석</UI> > <UI>멀티 트랜잭션 추적</UI> 메뉴에서 '차트 새창으로 띄우기' 버튼 클릭 시 발생하는 화이트 아웃 현상 수정

### 알림 메시지

<Status>New</Status> Application, Container, DB, Server 상품군에 신규 메뉴 <UI>사이트 맵</UI> > <UI>실험실</UI> > <UI>알림 메시지</UI> 추가

### 이벤트 설정

<Status>New</Status> <UI>경고 알림</UI> > <UI>이벤트 설정</UI> > <UI>메트릭스</UI> 메뉴에서 실험실의 <UI>알림 메시지 설정</UI> 메뉴 이동이 가능한 버튼 추가

### 인티그레이션

인티그레이션 아이템 설치 개수 변경(무제한 → 1개)에 따른 메뉴 진입 경로 및 디자인 수정

- <Status>Change</Status> <UI>인티그레이션 아이템</UI> 메뉴 위치를 <UI>관리</UI> 하위 메뉴로 이동, 하나의 인티그레이션 아이템만을 설치, 설정 가능하도록 수정

- <Status>Change</Status> 프로젝트 진입 시 인티그레이션 아이템 설치 현황에 따라 진입 경로 수정

  * 아이템이 설치된 경우: 아이템 종류에 맞는 대시보드로 이동
    * aws: <UI>Amazon CloudWatch</UI> > <UI>EC2</UI> 메뉴로 이동
    * ecs: <UI>Amazon ECS</UI> > <UI>Task</UI> 메뉴로 이동
    * 그 외: <UI>Flex 보드</UI> 메뉴로 이동

  * 아이템이 설치되지 않은 경우: <UI>관리</UI> > <UI>인티그레이션 아이템</UI> 메뉴로 이동

- <Status>Change</Status> 프로젝트 카드 디자인 수정

- <Status>Change</Status> 인트그레이션 아이템이 ECS인 경우: <UI>관리</UI> > <UI>인티그레이션 설치</UI> 메뉴에서 <UI>설치 안내</UI> 페이지 하단에 고정되었던 <UI>모니터링 서비스 활성화</UI> 버튼 위치를 설치 과정 내부로 이동

### 인티그레이션, 클라우드 모니터링

<Status>Deprecate</Status> <UI>관리</UI> > <UI>인티그레이션 아이템</UI> 메뉴에서 <UI>AWS CloudWatch</UI> 서비스 설정 시 <UI>전체 선택</UI> 체크 박스 제거

### 에이전트 설정

<Status>Fixed</Status> 에이전트 이름을 별칭으로 설정하면 서버 목록 화면에 반영되지 않는 문제 수정
