### 신규 기능 출시

Service 2.9.0 릴리스에 포함된 신규 기능을 소개합니다.

#### `Server`

<Status priority='1'>New</Status> 인프라 서버 운영에 필요한 정보를 에이전트가 자동 수집 및 저장, 사용자 정의 필드를 통해 추가한 정보를 조회하고 관리할 수 있는 <Cmdname sid="side_serverInventory" className="uitext" /> 메뉴 추가

:::note

- 이 기능은 <Cmdname sid="side_serverList" className="uitext" /> &gt; <Cmdname sid="side_serverInventory" className="uitext" /> 메뉴 경로에서 확인할 수 있습니다. 자세한 내용은 [다음 문서](../../server/server-inventory)를 참조하세요.

- 이 기능은 Server Linux/Windows 에이전트 **v2.6.6** 이상, AIX 에이전트 **v1.3.4** 이상, Solaris 에이전트 **v1.3.4** 이상에서 지원합니다.

:::

#### `Kubernetes`

- <Status priority='1'>New</Status> 노드(Node)에 대한 상세 정보를 확인할 수 있는 <Cmdname sid="side_nodeMap" className="uitext" /> 메뉴 추가

  :::note
  
  이 기능은 <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_nodeMap" className="uitext" /> 메뉴 경로에서 확인할 수 있으며, 다음 상세 정보를 제공합니다. 자세한 내용은 [다음 문서](../../kubernetes/node-map)를 참조하세요.

  - 노드별 리소스 정보

  - 개별 노드의 상태(CPU, Memory 상태)

  - 노드에 대한 환경 정보

  - 노드 내부에서 실행 중인 와탭 에이전트의 상태

  - 노드 관련 메트릭스 카테고리의 정상 수집 여부

  :::

- <Status priority='1'>New</Status> 시간의 흐름에 따른 노드 현황을 확인할 수 있는 <Cmdname sid="side_nodeTimeline" className="uitext" /> 메뉴 추가

  :::note

  이 기능은 이 기능은 클러스터 프로젝트 전용이며, <Cmdname sid="side_cluster" className="uitext" /> &gt; <Cmdname sid="side_nodeTimeline" className="uitext" /> 메뉴 경로에서 확인할 수 있습니다. 자세한 내용은 [다음 문서](../../kubernetes/node-timeline)를 참조하세요.

  :::

#### `Database`

- <Status priority='1'>New</Status> **CUBRID** <code className="my">V2</code> 서비스 시작

  :::note

  **CUBRID 모니터링** <code className="my">V2</code>의 새로운 기능에 대한 자세한 내용은 [다음 문서](../../cubrid/monitoring-intro#new-feature)를 참조하세요.

  :::

- <Status priority='1'>New</Status> SQL 스크립트를 실행해 결과를 확인할 수 있는 <Cmdname sid="side_dbScriptManager" className="uitext" /> 메뉴 추가

  :::note
