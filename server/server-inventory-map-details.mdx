<Cmdname sid="side_serverInventoryMap" className="uitext" /> 메뉴에서 선택한 서버 그룹의 자세한 정보를 ***상세 보기*** 화면에서 확인할 수 있습니다. **요약 보기** 패널 상단 오른쪽 위에 ![상세 보기 아이콘](/img/ico-detail-mode.svg) ***상세 보기*** 버튼을 클릭하세요. 상세 보기 내 서버 그룹 정보는 실시간으로 갱신됩니다.

* 상단 **시간 선택자**를 통해 그룹 데이터 조회 시점과 갱신 주기를 확인할 수 있으며 데이터 갱신을 중지 또는 재개할 수 있습니다. 

* 왼쪽 영역에서 **서버 그룹 카드** 및 **서버 목록**을 확인할 수 있습니다.

* 오른쪽의 개별 탭 영역에서 ***서버 인벤토리***, ***파일 시스템 정보***, ***메트릭스 정보***를 조회할 수 있습니다. 

## 서버 그룹 카드 및 목록

선택한 서버 그룹의 주요 요약 정보를 서버 그룹 카드 및 목록을 통해 확인할 수 있습니다. 

* 서버 그룹 카드 내 차트 영역에서 서버 위에 마우스를 올리면 선택한 서버의 요약 정보를 팝오버 형식으로 간략하게 확인할 수 있습니다. 

* 서버 그룹 카드 내 차트 영역과 서버 목록에서 조회를 원하는 대상 서버를 특정할 수 있습니다. 

* 서버는 최대 10개까지 선택할 수 있으며 대상 서버 선택 시 선택한 서버의 정보를 오른쪽 개별 탭에서 확인할 수 있습니다. 

## 서버 인벤토리 

선택한 서버의 상세 구성 정보를 <Cmdname sid="TTL08767" className="uitext" /> 탭에서 목록 형식으로 확인할 수 있습니다. 서버를 선택하지 않았을 경우 해당 서버 그룹 내 모든 서버에 대한 서버 구성 정보가 표시됩니다. <Cmdname sid="side_serverInventory" className="uitext" /> 메뉴에서 조회할 수 있는 정보와 구성이 동일합니다. 

### 필터 

<Cmdname sid="side_serverInventory" className="uitext" /> 메뉴에서 설정한 컬럼 목록을 기준으로 서버 필터 기능을 제공합니다.

* ***필터 키***: 필터는 <Cmdname sid="side_serverInventory" className="uitext" /> 메뉴에서 제공되는 기본 수집 컬럼과 사용자가 정의한 커스텀 컬럼을 기준으로 설정할 수 있습니다.

  * 기본 수집 컬럼: <Cmdname sid="side_serverInventory" className="uitext" />에서 자동 수집된 데이터 항목으로  ***서버 타입***, ***기본 그룹***, ***OS 타입***, ***OS 버전***, ***모델***, ***시리얼*** 컬럼 항목입니다.

  * 커스텀 컬럼: 사용자가 프로젝트별로 추가하여 관리하는 항목입니다.

* ***필터 조건***: 필터 조건은 일치, 불일치, 포함, 미포함과 같은 문자열 조건으로 설정할 수 있습니다. 모든 필터는 문자열 기반으로 적용되며, 숫자형 조건(예, `≥`, `≤`)은 지원하지 않습니다.

## 파일 시스템 정보 

왼쪽 서버 그룹 카드 또는 목록에서 선택한 서버의 파일 시스템 정보를 <Cmdname sid="TTL08666" className="uitext" /> 탭에서 목록 형식으로 조회할 수 있습니다. 서버를 선택하지 않았을 경우 해당 서버 그룹 내 모든 서버에 대한 파일 시스템 정보가 표시됩니다. 

|구분|설명|
|---|---|
|***Mount Point***|Mount된 파일 시스템의 이름을 표시합니다. Windows 환경에서는 드라이브 파티션의 이름을 나타냅니다.|
|***Total Space(GB)***|파일 시스템에 할당된 전체 저장 공간을 GB 단위로 표시합니다.|
|***Used Space(GB)***|파일 시스템에서 현재 사용 중인 저장 공간을 GB 단위로 표시합니다.|
|***Used(%)***|파일 시스템의 전체 저장 공간 대비 사용 중인 저장 공간의 비율을 백분율로 표시합니다.|
|***i-node Used(%)***|파일 시스템에 할당된 i-node 중 사용 중인 i-node의 비율을 백분율로 표시합니다.|
|***Free(GB)***|파일 시스템에서 현재 사용 가능한 여유 공간을 GB 단위로 표시합니다.|

:::note 

**i-node**

i-node는 파일 시스템에서 파일과 디렉터리에 대한 정보를 저장하는 공간으로, 이 사용률이 100%가 되면 ***Used(%)***가 100%가 아니더라도 더 이상 파일을 쓸 수 없습니다.

:::

### 필터

## 메트릭스 

왼쪽 서버 그룹 카드 또는 목록에서 선택한 서버의 성능 지표 차트를 <Cmdname sid="metrics" className="uitext" /> 탭에서 조회할 수 있습니다. 서버를 선택하지 않았을 경우 해당 서버 그룹 내 상위 50대 서버에 대한 성능 지표 차트가 표시됩니다. 

|이름|설명|비고|
|---|---|---|
|***[CPU] User + Sys (%)***|전체 CPU 사용률 중 사용자 모드(User)와 커널 모드(Sys)에서의 사용률 합계를 표시하며, 전체 CPU 자원의 활용도를 나타냅니다.|-|
|***[CPU] User (%)***|사용자 모드에서 실행 중인 프로세스가 차지하는 CPU 비율입니다.|-|
|***[CPU] Sys (%)***|커널 모드에서 실행 중인 프로세스가 차지하는 CPU 비율입니다.|-|
|***[CPU] Wait (%)***|CPU가 디스크나 네트워크와 같은 I/O 작업을 기다리며 유휴 상태에 있는 시간 비율입니다.|-|
|***[System] Runqueue***|실행 준비가 되어 있지만 아직 CPU를 할당받지 못한 프로세스(또는 Thread)의 수입니다.|-|
|***[System] Blockqueue***|I/O, 페이징 등을 위해 대기열에 있는 프로세스(또는 Thread) 수를 의미합니다.|Windows 제외|
|***[System] Forks***|초당 새로운 프로세스를 생성하는 작업 횟수를 표시합니다.|Windows 제외|
|***[System] Interrupts***|초당 CPU가 하드웨어 인터럽트를 처리한 횟수를 표시합니다.|-|
|***[System] SystemCall***|초당 커널이 시스템 호출을 처리한 횟수를 표시합니다.|Linux 제외|
|***[System] ContextSwitch***|초당 프로세스 간의 전환이 발생한 횟수를 표시합니다.|-|
|***[MEM] Used+Cache(%)***|전체 메모리 중 Used 메모리와 Cached 메모리의 합계 비율을 표시합니다.|Windows는 <UI>[MEM] Used(%)</UI>로 표시|
|***[MEM] Used(%)***|메모리가 사용자 프로세스나 커널에 의해 사용된 메모리의 사용률을 의미하며 버퍼 및 캐시 영역은 제외하고 표시합니다.|-|
|***[MEM] Cache(%)***|자주 접근되는 파일이나 데이터를 빠르게 제공하기 위해 캐시된 메모리의 사용 비율을 표시합니다.|Windows 제외|
|***[Swap] Used(%)***|현재 사용 중인 Swap의 크기를 백분율로 표시한 값입니다.|-|
|***[MEM] SwapIn***|디스크의 스왑(페이징 스페이스) 영역에서 메모리로 옮겨간 페이지 수를 표시합니다.|-|
|***[MEM] SwapOut***|메모리에서 디스크 스왑(페이징 스페이스) 영역으로 옮겨간 페이지 수를 표시합니다.|-|
|***[MEM] FilesystemIn***|파일 시스템의 읽기 작업에 따라 메모리에 로드한 페이지 수를 표시합니다.|-|
|***[MEM] FilesystemOut***|파일 시스템의 쓰기 작업에 따라 디스크에 기록한 페이지 수를 표시합니다.|-|
|***[NET] Rx+Tx (bps)***|수신(Rx) 및 송신(Tx)되는 데이터의 합을 bps 단위로 표시한 값입니다.|-|
|***[NET] Rx+Tx (pps)***|수신(Rx) 및 송신(Tx)되는 패킷의 합을 초당 패킷(pps) 단위로 표시한 값입니다.|-|
|***[Disk] Read + Write (bytes)***|디스크 읽기와 쓰기 데이터의 합계를 바이트 단위로 표시합니다.|-|
|***[Disk] Read + Write (IOPS)***|디스크 읽기와 쓰기 작업의 횟수를 IOPS(Input/Output Operations Per Second) 단위로 표시합니다.|-|
|***[Process] TotalCount***|현재 전체 프로세스 개수를 표시합니다.|-|
|***[Thread] TotalCount***|현재 전체 Thread 개수를 표시합니다.|-|
|***[Defunct] TotalCount***|현재 defunct 프로세스 개수를 표시합니다.|-|
