데이터베이스 지표(메트릭스)는 데이터베이스의 작업량과 리소스 사용량을 포괄적으로 수집합니다. 와탭 모니터링 서비스는 지속적으로 지표를 수집하여 데이터베이스의 병목 현상을 확인하고, 성능 문제의 근본 원인을 식별하는 데 도움이 되는 데이터를 제공합니다.

| 지표 | 설명 |
| --- | --- |
| `active sessions` | 쿼리를 실행 중인 세션 수 |
| `block disk read` | 디스크로부터 block을 읽은 횟수 |
| `block gets (CRX)` | CR block examine을 요청한 횟수 |
| `block updates` | Redo 로그를 남기고 block을 수정한 횟수 |
| `connect_error` | 접속 에러가 발생한 수 |
| `consistent block gets` | CR block을 요청한 횟수 |
| `consistent multi gets` | Multi CR block을 요청한 횟수 |
| `cpu` | CPU 사용률(%) |
| `cpu_cores` | CPU 코어 수 |
| `current block gets` | Current block을 요청한 횟수 |
| `current block gets - no wait` | Current block을 기다리지 않고 얻은 횟수 |
| `dbwr write` - OS | DBWR가 OS에 write를 요청한 횟수 |
| `execute count` | SQL 문(statement)을 실행한 총 호출 수 |
| `fatal count` | 치명적인 오류가 발생한 횟수 |
| `iowait cpu` | CPU가 입출력을 대기하는데 사용한 시간 비율 |
| `lock wait sessions` | 현재 row lock을 획득하기 위해 대기 중인 수 |
| `logical reads` | 메모리에서 읽은 데이터 블록 수, `db block gets`와 `consistent gets`의 합 |
| `logons cumulative` | 인스턴스가 시작된 후 총 로그온 수 |
| `long running sessions` | 오랫동안 실행 중인 세션 수 |
| `multi block disk read` | 디스크로부터 multi block을 읽은 횟수 |
| `parse count (hard)` | 총 구분 문석 호출 수(실제 구문 분석) |
| `parse count (total)` | 총 구문 분석 호출 수(`hard`, `soft`, `describe`) |
| `physical reads` | 디스크에서 읽은 데이터 블록 수 |
| `pq sessions` | 병렬 처리를 수행 중인 세션 수 |
| `sorts (memory)` | 메모리에서 수행된 sort 작업 수 |
| `sys cpu` | session에 의해 사용된 `cpu time`의 합 |
| `the number of user commits performed` | `sys cpu` 사용량(%) |
| `total elapse time` | 데이터베이스 인스턴스가 마지막으로 다시 시작된 이후 모든 요청의 누적 총 경과 시간 |
| `total sessions` | 모든 세션 수 |
| `user cpu` | 사용자가 실행하는 쿼리나 프로시저 등과 같은 작업을 수행하는 CPU 사용량(%) |
| `wait sessions` | 이벤트를 대기하고 있는 세션 수 |
| `warning count` | 모든 경고 수 |
