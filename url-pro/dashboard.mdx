와탭은 세 가지 대시보드를 제공해 URL 모니터링 현황을 여러 관점에서 확인할 수 있습니다.

## 카테고리 현황 대시보드

URL을 카테고리별로 구분해 실시간 데이터를 조회할 수 있습니다. 와탭 URL 모니터링 에이전트가 각 사이트에 주기적으로 접속해 정상 여부를 수집하므로, 운영자는 브라우저로 직접 점검할 필요가 없습니다.

<ImgLang img='url-pro/category-dashboard.png' desc='카테고리 현황 대시보드' />

### **카테고리 상태 현황**

카테고리에 등록된 URL의 상태 정보를 제공합니다. 카테고리 이름, 전체 URL 수, 정상 URL 수, 비정상 URL 수를 건수와 차트로 표시합니다. 

카테고리 위젯을 클릭하면 카테고리 상세가 표시됩니다.

### **카테고리 상세**

선택한 카테고리에 등록된 URL별 상태 현황 목록을 표시합니다. 상태, 페이지 명, 메시지, 추이에 대한 정보를 확인할 수 있습니다. 

목록을 클릭하면 해당하는 **URL 상세** 정보가 표시됩니다.

## 이슈 현황 대시보드

URL을 이슈별로 실시간 데이터를 조회할 수 있습니다. 와탭의 URL 모니터링 에이전트가 해당 사이트에 접속해, 정상 접근 여부를 점검합니다.

### **이슈 상세**

이슈에 등록된 URL의 상태 현황 목록을 표시합니다. 상태, 페이지 명, 메시지, 추이에 대한 정보를 확인할 수 있습니다. 목록을 클릭하면 해당하는 **URL 상세** 정보가 표시됩니다.

## 장애 현황 대시보드

<ImgLang img='url-pro/error-dashboard.png' desc='장애 현황 대시보드' />

실시간 장애 발생 URL 목록과 장애 상태 지속 URL 정보를 보여줍니다. 목록을 클릭하면 해당 **URL 상세** 정보가 표시됩니다.

- 실시간 장애 발생 URL: 장애 조건에 부합하지만 설정한 발생 횟수에 미달한 URL 목록
- 장애 상태 지속 URL: 설정된 횟수만큼 장애가 발생하여 장애 지속 상태로 분류된 URL 목록으로 타이틀이나 로케이션 변경의 경우 **원인 확인** 버튼을 클릭하여 사용자 확인 후 **정상 처리** 버튼을 클릭하여 목록에서 제거


## 대시보드 공통 기능

### 실시간 데이터 조회

- 실시간 모드: 30초 간격 자동 갱신
- 일시정지 모드: 자동 갱신을 정지

### 필터

상단 옵션바의 **필터**에서 쿼리 언어 선택 및 검색 조건 입력해 필터를 걸 수 있습니다.

### 검색 쿼리 문법

WhaTap 로그 검색 쿼리는 자동 완성 기능을 제공하며, 간단한 단어 입력만으로도 문제의 로그를 신속하게 검색할 수 있습니다. and, or 등 다양한 문법을 활용해 다양한 검색 조건을 입력할 수 있습니다.

- 예, 검색 필터에 `error`만 입력하여 content(로그 본문) 검색할 수 있습니다.
    1. 상단 옵션 바에서 필터 아이콘을 클릭한 후, 쿼리 검색창에서 필드(선택한 시간과 카테고리에 맞는 목록 노출)를 선택하세요.
    2. 연산자를 선택하세요.
        
        a. 기본: `:`, `: *` , `and`, `or`
        
        b. 선택한 필드가 `Number`인 경우(기본 + `<=`, `>=` , `<`, `>`)
        
        c. 선택한 필드가 `String`인 경우(기본)
        
    3. 값(선택한 시간, 카테고리, 필드에 맞는 목록 노출)을 선택하세요.

### **검색 인덱스 필드 타입**

지원 필드 타입은 `Number`, `String`, `Boolean` 세 가지 유형만 존재합니다.

- `String`: Long, double, float 타입의 숫자 값
- `Number`: 이메일 본문과 같은 전체 텍스트
- `Boolean`: True 또는 False 값

WhaTap log search query 문법에 대한 자세한 내용은 [다음 문서](https://docs.whatap.io/log/log-search-query)를 참고하세요.

### **Lucene**

apache lucene 8.11.1 버전 오픈소스 문법을 그대로 사용하세요. 단, 자동 완성 기능을 제공하지 않습니다.

Lucene의 QueryParser 클래식 패키지에 대한 자세한 내용은 [공식 문서](https://lucene.apache.org/core/8_11_1/queryparser/org/apache/lucene/queryparser/classic/package-summary.html#Overview)를 참고하세요.
