WhaTap은 카테고리 현황, 이슈 현황, 장애 현황 세 가지 대시보드를 제공합니다.
이를 통해 URL 모니터링 현황을 다양한 관점에서 확인할 수 있습니다.

대시보드 상단에는 시간 설정, 정렬, 필터 옵션이 있습니다.
단, 장애 현황 대시보드는 정렬 기능을 지원하지 않습니다.

## 대시보드 공통 기능

### 1. 시간 설정

상단 옵션바의 **타임셀렉터**를 통해 검색 시간 범위를 설정할 수 있습니다.

    * 실시간 모드: 30초 간격 자동 갱신
    * 일시정지 모드: 자동 갱신을 정지 후 조회 범위 지정

### 2. 정렬 

대시보드에 표시되는 데이터를 원하는 기준으로 정렬할 수 있습니다.

    * 이름순
    * 총 개수 순
    * 비정상 개수 순
    * 비정상 비율 순

### 3. 필터

상단 옵션 바의 필터 영역에서 검색 쿼리를 입력해 데이터를 필터링할 수 있습니다.

    * URL 모니터링에서는 카테고리, 페이지명, 이슈, URL 필드에 대해 필터를 적용할 수 있습니다.

    * 각 필드 끝에는 해당 필드의 데이터 타입 아이콘이 표시됩니다.
    
        * ![숫자 아이콘](/img/ic-slack-color.svg): Number
        * ![텍스트 아이콘](/img/ic-text-color.svg): String
        * ![활성 아이콘](/img/ic-boolean-color.svg): Boolean

    * 데이터 타입에 맞는 등의 연산자를 선택 합니다. 
        
    * 여러 조건은 `and`, `or` 연산자를 사용해 설정할 수 있습니다.


#### 필터 적용하기

    1. 상단 옵션 바에서 필터 영역을 클릭합니다.

    1. 쿼리 검색창에서 필드(카테고리, 페이지명, 이슈, URL)를 선택합니다.

    1. 연산자를 선택합니다.

        * 기본: `:`, `: *` , `and`, `or` 
        * String 필드: 기본 연산자 이용 
        * Number 필드: 기본 연산자 + `<=`, `>=` , `<`, `>`

    1. 값을 입력합니다.

## 카테고리 현황 대시보드

URL을 카테고리별로 구분해 실시간 상태를 조회할 수 있습니다.  
WhaTap URL 모니터링 에이전트가 각 사이트에 주기적으로 접속해 정상 여부를 수집하므로 운영자가 직접 점검할 필요가 없습니다.

<ImgLang img='url-pro/category-dashboard.png' desc='카테고리 현황 대시보드' />

### 카테고리 상태 현황
카테고리에 등록된 URL의 상태 정보를 제공합니다.  
- 카테고리 이름, 전체 URL 수, 정상 URL 수, 비정상 URL 수를 건수 및 차트로 표시합니다.  
- 카테고리 위젯을 클릭하면 **카테고리 상세** 화면이 열립니다.  

### 카테고리 상세
선택한 카테고리에 등록된 URL별 상태 현황을 목록으로 표시합니다.  
- 상태, 페이지명, 메시지, 추이 확인 가능  
- URL을 클릭하면 해당 **URL 상세** 화면을 확인할 수 있습니다.

## 이슈 현황 대시보드

URL을 이슈별로 구분해 실시간 상태를 조회할 수 있습니다.  
WhaTap URL 모니터링 에이전트가 각 사이트에 접속해 정상 접근 여부를 점검합니다.

### 이슈 상세
이슈에 등록된 URL의 상태 현황을 목록으로 표시합니다.  
- 상태, 페이지명, 메시지, 추이 확인 가능  
- URL을 클릭하면 해당 **URL 상세** 화면을 확인할 수 있습니다.

## 장애 현황 대시보드

<ImgLang img='url-pro/error-dashboard.png' desc='장애 현황 대시보드' />

실시간 장애 발생 URL과 장애 상태 지속 URL을 확인할 수 있습니다.  
- 목록 클릭 시 해당 **URL 상세** 화면이 열립니다.  

**표시 항목**  
- **실시간 장애 발생 URL**: 장애 조건에 부합하지만 발생 횟수에 미달된 URL 목록  
- **장애 상태 지속 URL**: 설정된 횟수만큼 장애가 발생하여 지속 상태로 분류된 URL 목록  
  - 타이틀이나 로케이션 변경 시, **원인 확인** 버튼으로 검토 후 **정상 처리** 버튼을 클릭해 목록에서 제거할 수 있습니다.

## URL 상세 화면

선택한 URL의 상세 모니터링 데이터와 이력을 확인할 수 있습니다.  

<ImgLang img='url-pro/url-detail.png' desc='URL 상세 화면' />

- **기본 정보**: URL, 페이지명, 모니터링 주기, 상태, 카테고리, 이슈  
- **허용 목록**: 정상 처리된 페이지명/URL  
- **URL 상태 추이**: 시간대별 응답 시간(elapsed)과 장애 상태 추이  
- **작업 및 관리 내역**: 사용자 기록 관리  
- **로그 조회**: URL 접속 시도에 대한 상세 로그

## 장애 판정 및 처리

### 장애 단계
1. **비정상 (abnormal)**  
   - 장애 조건에 부합했으나 발생 횟수 미달  
   - 경고 상태 (주황색)  
2. **장애 (error)**  
   - 장애 조건 및 횟수 충족  
   - 알림 발생 상태 (빨간색)  

### 정상 처리
일부 장애 유형(페이지 제목 변경, URL 변경)은 정상적인 변경일 수 있습니다.  
사용자가 확인 후 **정상 처리**를 수동으로 수행할 수 있습니다.  

- 정상 처리된 변경 사항은 **허용 목록**에 추가됩니다.  

  <ImgLang img='url-pro/url-whitelist.png' desc='URL 상세 화면' />

- 이후 동일한 변경은 장애로 감지되지 않습니다.
