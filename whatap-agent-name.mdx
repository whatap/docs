## 에이전트 이름(oname), 에이전트 아이디(oid)

에이전트 이름(oname)은 모니터링 대상을 [와탭 모니터링 서비스](https://service.whatap.io)에서 식별하기 위한 설정 옵션 중 하나입니다. 하나의 프로젝트에 여러 에이전트를 연결한 경우 에이전트 이름으로 모니터링 대상을 구분할 수 있습니다. 예를 들어 애플리케이션 대시보드에서는 화면 위에서 에이전트 이름을 확인할 수 있습니다.

<ImgLang img='apm-agent-name.png' desc='' />

에이전트 이름은 모니터링 대상 서버로부터 수집한 정보를 활용합니다. 기본 활용 정보는 모니터링 대상 서버의 종류, IP 주소 등을 조합해 고유 식별자로 사용합니다. 필요에 따라 사용자가 설정한 명칭을 이용하거나 패턴을 변경해 고유한 값으로 변경해 이용합니다. 

에이전트의 이름은 반드시 고유한 값이어야 합니다. 애플리케이션 서버로부터 추출한 정보를 활용하는 이유는 애플리케이션 서버 정지 또는 네트워크 단절, 에이전트 문제로 인한 수집 서버와 에이전트의 통신 두절 상태가 복구되었을 경우, 재접속된 에이전트로부터 송신되는 정보가 기존 에이전트로부터 송신된 정보와의 연속성을 유지하기 위해서입니다.

:::note

* 에이전트 이름은 프로젝트 단위로 고유해야 합니다.

* 에이전트 아이디/이름을 변경하면 이전 데이터와 연결되지 않습니다.

* 와탭 모니터링 서비스에서는 모니터링 대상을 오브젝트 또는 에이전트로 혼용해서 사용하기도 합니다.

:::

에이전트 아이디(oid)는 에이전트 이름(oname)을 토대로 생성됩니다. 

에이전트 이름(oname)과 에이전트 아이디(oid)는 모니터링 대상의 모든 정보를 조회할 때 이용됩니다. 트랜잭션, 히트맵, 메트릭스 등 다양한 데이터를 조회하고 데이터에 해당하는 모니터링 대상이 무엇인지 확인하는데 사용됩니다.

## 상품별 에이전트 이름

사용자가 이용하는 모니터링 상품에 따라 에이전트에 대한 이름은 다를 수 있습니다. 이는 상품의 특성을 더 잘 표현하기 위함입니다. 다음 표를 통해 에이전트를 부르는 이름이 어떻게 다른지 확인하세요.

| 상품 | 명칭 |
| -- | -- |
| Application, Server | 에이전트 이름 / 오브젝트 이름 |
| Kubernetes | 마스터 에이전트 이름 / 노드 에이전트 이름 |
| Database | 인스턴스 이름 / 에이전트 이름 |
